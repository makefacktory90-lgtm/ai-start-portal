# Ответ на вопросы ТВ-21: практика, клише, данные, вред изданию

## 1. Практика не переносится целиком в домашнюю работу

Практика предусмотрена внутри занятий. Мы не превращаем встречу в марафон вопросов и ответов, но в каждом воркшопе есть короткое действие руками:

```text
объяснение рамки
-> демонстрация
-> короткая практика в аудитории
-> разбор 2-3 ответов
-> межсессионное закрепление на своем материале
```

Для первого занятия:

| Блок | Время | Что делает участник |
|---|---:|---|
| Role mapping | 7 мин | выбирает свой участок работы и ближайший пайплайн |
| AI smell test | 8 мин | находит шаблонные места в AI-тексте |
| Starter prompt | 12 мин | прогоняет общий материал через безопасный prompt |

Межсессионная работа нужна не вместо практики, а чтобы участник применил тот же маршрут на своем материале.

## 2. Типичные клише и шаблонные ошибки LLM

В первом воркшопе это вводится как редакторский smell test.

| Ошибка | Как распознать | Как избегать |
|---|---|---|
| Универсальные фразы | текст можно поставить в любой материал | заменить на факт, цитату, место, героя, число |
| Раздувание события | AI делает событие крупнее, чем оно есть | сверить масштаб с источником |
| Канцелярская гладкость | одинаковый ритм, симметричные списки, уверенные обобщения | просить варианты с evidence и отмечать риск |
| Ложная полнота | текст звучит готовым, но фактов не хватает | вводить блок "что нужно проверить" |
| Потеря голоса редакции | output похож на общий пресс-релиз | использовать ToV / Style Graph и редакторский gate |

Главная формулировка:

> Нейротекст видно не по факту использования AI, а по пустой гладкости: нет источника, нет наблюдения, нет редакторской проверки.

## 3. Защита внутренних данных

Это не вопрос "верим сервису или нет". Это вопрос data gate.

| Уровень | Что можно делать |
|---|---|
| Green | публичные материалы, утвержденные тексты, открытые источники |
| Yellow | внутренние черновики без персональных данных и коммерческой тайны, если это разрешено политикой |
| Red | персональные данные, доступы, договоры, непубличные документы, внутренние расследования |

Позиция:

> Безопасность - это не обещание сервиса, а правило редакции: что можно загружать, куда, в каком аккаунте, с какими настройками и кто отвечает за проверку.

OpenAI указывает, что данные ChatGPT Business / Enterprise по умолчанию не используются для обучения моделей, но это не отменяет внутреннюю политику редакции и список запрещенных данных.

## 4. Может ли AI навредить изданию

Да, если использовать его как дешевый генератор одинакового контента без опыта, проверки и редакторского решения.

Google Search Central формулирует принцип так: важны качество, оригинальность, полезность и E-E-A-T; использование автоматизации, включая AI, для манипуляции поисковой выдачей нарушает spam policies.

Для курса это превращается в publication gate:

| Gate | Вопрос |
|---|---|
| Original | есть собственный факт, герой, место, наблюдение или опыт редакции? |
| Verified | ключевые факты проверены? |
| Useful | материал отвечает читателю, а не просто заполняет канал? |
| Owned | есть человек, который отвечает за выпуск? |

Формулировка:

> Опасен не AI как инструмент. Опасен дешевый неуникальный output, который не добавляет редакционной ценности и не проходит human gate.

## 6. Что берем из подхода Тони Жуковой без personal OS

Берем не контент и не personal OS, а структуру подачи:

- mindset shift: не "вот промпты", а "как теперь устроена работа";
- tools as layers: инструменты показываются по участкам работы;
- insights: короткие принципы, которые остаются после занятия;
- checklist: участник понимает, что сделать руками;
- hub: удачные приемы сохраняются и становятся командной памятью.

Адаптация под AI Start:

```text
OS роли
-> OS редакции
-> OS компании
-> quality gates
-> hub update
```
