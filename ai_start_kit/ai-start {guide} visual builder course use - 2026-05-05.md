# Visual Builder: как используем в курсе

## 1. Что это

`visual-builder.html` - локальный visual lab для блока 19 мая.

Путь:

```text
output/ai_start_course_site/visual-builder.html
```

Локальный URL при поднятом сервере:

```text
http://127.0.0.1:4317/visual-builder.html
```

## 2. Главная роль в курсе

Это не "генератор красивых картинок" и не набор готовых макетов для клиента.

Это учебный слой, который помогает редакции увидеть принцип работы с визуалом:

```text
материал
-> смысл
-> формат
-> редактируемый шаблон
-> visual brief
-> AI asset только как фон / метафора
-> human quality gate
```

Мы показываем участникам варианты инструментов и саму логику:

- как отделить факты от визуального asset;
- как оставить публикационный текст редактируемым;
- как выбрать формат под задачу;
- как перенести принцип в их реальные TV cover, title, intertitle, interactive quad и другие редакционные форматы.

Визуал в редакции должен держаться на:

- формате;
- сетке;
- слоях;
- проверяемом тексте;
- ограничениях по фактам;
- читаемости;
- повторяемом шаблоне.

Реальные форматы клиента не додумываем до получения материалов. После 10 мая анализируем присланные примеры и подставляем их в готовую рамку.

## 3. Почему это важно для ТВ-21 и "Северного города"

Редакциям нужно быстрее и стабильнее выпускать:

- обложки;
- карточки;
- quote cards;
- title cards для вертикальных видео;
- intertitles;
- interactive quad / экранные раскладки, если они реально используются;
- простые визуальные анонсы.

Но если поручить это image model целиком, получаем:

- текст внутри картинки, который нельзя быстро исправить;
- неверные числа / цитаты;
- странные артефакты;
- красивую, но непроверяемую графику;
- каждый раз новый стиль.

Наша логика:

> AI помогает предложить структуру и asset. Шаблон держит публикационный текст и факты.

Мы не обещаем сделать каждый формат руками. Мы даем принцип инструмента, prompt logic, quality gate и место в hub, куда после клиентских материалов попадут их реальные примеры.

## 4. Как показывать на занятии 19 мая

### Demo 1. Формат

Показать переключение:

- Story 1080x1920;
- Square 1080x1080;
- YouTube 1920x1080;
- Banner 1920x640.

Смысл:

> Один материал должен жить в разных форматах, но факты и структура должны оставаться управляемыми.

### Demo 2. Layout

Показать:

- left;
- center;
- split;
- poster;
- minimal.

Смысл:

> Layout - это решение о внимании: где главный смысл, где подпись, где служебная информация.

### Demo 3. Content layer

Поменять:

- kicker;
- title;
- subtitle;
- name plate.

Смысл:

> Публикационный текст должен быть редактируемым слоем. Не запекаем факты в AI-картинку.

### Demo 4. Metaphor / AI background prompt

Показать metaphor:

- industrial layers;
- editorial signal radar;
- pipeline routes;
- archive cards.

Смысл:

> Генеративный asset может помогать образу, но не должен нести фактическую нагрузку.

### Demo 5. Export

Показать PNG/SVG export как prototype.

Смысл:

> Финальный production может жить в Figma/Canva/монтажке, но мышление о слоях и QA одинаковое.

## 5. Как связать с видео-стеком

Для вертикального видео visual builder показывает слой:

```text
clip candidate
-> hook
-> subtitle chunks
-> title card / cover
-> platform export
```

То есть для 19 мая у нас два связанных маршрута:

### Video route

```text
long video -> transcript -> clip candidates -> vertical edit
```

### Visual route

```text
clip / material -> cover / title card / visual brief
```

Visual builder помогает объяснить второй маршрут.

## 6. Какой output должен получить участник

После блока участник должен уметь собрать не финальную дизайн-систему, а draft visual brief под свой материал:

```text
1. Формат
2. Главный смысл
3. Текстовые слои
4. Что нельзя искажать
5. Визуальная метафора / asset prompt
6. Где будет редактируемый текст
7. Quality gate перед публикацией
```

Для клиентских материалов после 10 мая добавляем второй слой:

```text
1. Реальный формат клиента
2. Что в нем является фактом / текстом / визуальным asset
3. Где AI помогает
4. Где нужен шаблон редакции
5. Какой prompt и gate сохраняем в AI-hub
```

## 7. Мини-задание

Дать участникам материал и попросить:

```text
1. Выберите формат: story / square / cover / banner.
2. Сформулируйте title до 7 слов.
3. Напишите subtitle.
4. Выберите visual metaphor.
5. Сгенерируйте AI background prompt.
6. Отдельно выпишите, какие факты нельзя запекать в картинку.
7. Проверьте по visual gate.
```

## 8. Что фиксируем для внутреннего согласования

Смысл:

> Мы не продаем отдельный "инструмент для графики" и не делаем пачку макетов за редакцию. Мы показываем принцип: шаблон держит факт и текст, AI помогает с asset, метафорой, вариантами и быстрым visual brief. Это должно стыковаться с вертикальным видео: cover, title card, intertitle, subtitle layer.

До получения материалов клиента не задаем дополнительных вопросов. Если обсуждаем с видео/дизайн-специалистами, то согласуем только рабочую логику:

- какие типы форматов показываем как категории: cover, title, intertitle, quote, interactive quad;
- какие функции инструментов нужны: шаблон, captions, asset generation, export, QA;
- где production обычно живет: Figma, Canva, CapCut, Premiere, DaVinci или текущий редакционный инструмент;
- как объяснить участникам, что AI не заменяет редакционный шаблон;
- какие места в hub оставляем под клиентские примеры после 10 мая.

После 10 мая:

```text
клиентские материалы
-> разбор реальных визуальных форматов
-> привязка к pipeline 02/03
-> замена synthetic examples
-> обновление prompts/checklists в AI-hub
```

## 9. Что надо доработать в visual builder

Минимально:

- добавить preset "vertical clip cover";
- добавить preset "intertitle";
- добавить preset "quote from transcript";
- добавить visual gate прямо в правую панель;
- добавить кнопку / блок "что нельзя запекать в картинку";
- переименовать `AI Start Visual Lab` в более учебное `AI Start Visual Lab`.

Не надо сейчас:

- превращать его в полноценный Figma;
- делать backend;
- делать авторизацию;
- делать автогенерацию через API.

## 10. Формулировка в одну строку

Visual Builder - это учебный prototype, который показывает редакции:

```text
графика = смысл + формат + редактируемые слои + проверка,
а не "нейросеть, сделай красиво".
```
