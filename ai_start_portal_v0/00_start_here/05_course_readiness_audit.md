# Course readiness audit

Статус: методологический аудит v0.1  
Цель: довести курс до состояния, где его можно показывать структурному партнеру и клиенту без ощущения "мы накидали умных тем".

## Короткий вывод

Курс уже держится на правильной архитектуре:

- пять рабочих пайплайнов редакции;
- уровни выхода Base / Opti / Advanced / Scout;
- web-decks 01-10;
- hub, gates, prompt library, recordings;
- учет вопросов клиента про клише, безопасность и вред AI-контента.

Но курсу еще не хватает жесткого операционного слоя. Сейчас местами видно "о чем урок", но не всегда видно "что участник сделал руками, по какому материалу, каким промптом, как мы поняли, что результат годный, и куда он лег после занятия".

## Что уже хорошо

### 1. Правильная рамка для взрослых практиков

Курс не продает "магические нейросети". Он говорит:

```text
source
-> AI draft
-> human gate
-> hub
```

Это правильная рамка для телевизионных редакций. Люди не студенты, им не надо объяснять, что такое текст или монтаж. Им надо показать, где AI забирает первый проход, черновую сортировку, варианты, расшифровку, repurpose, visual prompt, но не забирает вкус, ответственность и решение.

### 2. Клиентские вопросы учтены

Закрыты три главных возражения:

- клише и пластиковые ошибки AI;
- безопасность внутренних данных;
- риск вреда изданию из-за шаблонного / неуникального / непроверенного контента.

Но эти ответы надо не только проговорить. Они должны быть в упражнениях:

- клише ловим в Deck 02 через Deplasticizer;
- безопасность проверяем в каждом prompt через source classification;
- вред изданию закрываем publication gate до финального текста / клипа / карточки.

### 3. Есть структура после вебинара

Есть слот:

```text
recording_link.md
transcript.md
summary.md
questions_and_answers.md
homework.md
```

Это нужно оставить. Но надо назначить SLA и владельца публикации, иначе записи опять расползутся по чатам.

## Что надо добить до уровня "можно показывать"

### Priority 1. Lesson contracts

Для каждого урока нужен контракт:

- с чем уходим;
- что делаем на занятии;
- какой входной материал;
- какой prompt;
- какой output;
- какой quality gate;
- что идет в hub;
- что считается "зачтено";
- что делать Base / Opti / Advanced.

Файл:

- `../08_presentations/ai-start {methodology} lesson contracts - 2026-05-09.md`

### Priority 2. Prework для участников

Перед каждым занятием участник должен видеть:

- что открыть;
- какой материал подготовить;
- можно ли использовать клиентский материал или нужен обезличенный;
- какой аккаунт / сервис нужен;
- что делать, если сервиса нет.

Без этого практика превращается в "сейчас пять минут все ищут ссылку". А потом еще десять минут кто-то не залогинен. Классика жанра, но нам эта классика не нужна.

### Priority 3. Ролевые маршруты

Надо явно развести:

- редактор;
- корреспондент;
- монтажер;
- SMM / Telegram;
- дизайнер / визуальный редактор;
- выпускающий / руководитель.

Один и тот же урок должен иметь разные выходы для ролей. Например, в видеоуроке монтажер забирает rough-cut logic и gate, а редактор забирает clip selection и риск контекста.

### Priority 4. Client material map

Материалы ТВ-21 уже разобраны, но надо жестче привязать их к занятиям:

- новости "Без интернета" / "Мормлеки" -> Source Map, факты, риск усиления;
- "О мой блог" -> разговорный стиль, voice, юмор, spoken script;
- Telegram -> repurpose и editorial contour;
- визуалы -> visual brief, card structure, style extraction;
- видео -> clip candidates, transcript map, монтажный gate.

### Priority 5. Prompt library должна стать не индексом, а стартовым набором

Сейчас `master_prompt_index.md` описывает типы промптов. Для занятия нужно больше:

- готовый prompt;
- когда использовать;
- какой input вставлять;
- какой output ждать;
- какой gate применить;
- типичный failure;
- как чинить.

### Priority 6. Quality gates должны быть в каждом уроке

Каждый урок должен кончаться не "получили красивый результат", а "прогнали через gate".

Минимальные gates:

- source gate;
- factual gate;
- tone / plastic gate;
- security gate;
- publication risk gate;
- format gate.

### Priority 7. Нужны speaker scripts для уроков 02-10

Deck 01 уже ближе к готовому сценарию. Остальным пока не хватает "что я говорю и что показываю".

Формат:

```text
тайминг
что говорю
что показываю
что вставляю в prompt
что участники делают
что фиксируем в hub
```

### Priority 8. Визуальный стиль уроков 03-10

Deck 03-10 еще используют старые квадратные robot images. Это надо заменить на взрослые cinematic scenes в стиле art direction keyframe:

- не игрушечный робот;
- не карточка с персонажем;
- редакционная sci-fi среда;
- Moebius / Daniel Isles / Dirty Robot / Nolan mood;
- конкретный рабочий сюжет урока.

### Priority 9. Tool radar надо встроить в уроки

Tool radar есть отдельно, но в каждом уроке должен быть блок:

```text
что можно использовать уже сейчас
что можно смотреть продвинутым
что пока сыро
что запрещено для внутренних данных
```

Особенно для видео:

- transcript tools;
- auto clipping;
- pause removal;
- multicam;
- Premiere / Resolve automation;
- MCP / agent scenarios.

### Priority 10. Adoption dashboard

Финал курса должен иметь не просто "ура, kit готов", а 30-дневный контур:

- сколько материалов прогнали через pipeline;
- сколько outputs принято / отклонено;
- сколько времени сэкономлено;
- какие ошибки повторились;
- какие prompts / gates обновили;
- кто owner следующего шага.

## Что я бы не делал сейчас

Не надо сейчас расписывать все лекции как учебник на 80 страниц. Это убьет живость.

Надо сделать жесткий backbone:

- contracts;
- prework;
- prompts;
- gates;
- speaker run-of-show;
- hub output.

А уже потом усиливать примерами, артами и конкретными демками.

## Минимум до показа партнеру

Чтобы показать партнеру в 12:00 и выглядеть собранно:

1. Главная портала должна показывать весь курс и все 10 decks.
2. Должен быть виден `lesson contracts` как методологический центр.
3. Deck 01 и Deck 02 должны выглядеть как эталон стиля.
4. Deck 03-10 могут быть backbone, но честно обозначены как practical draft.
5. На каждый урок должен быть понятен output.
6. Должно быть видно, что вопросы клиента закрыты не ответом, а практикой.

## Главная формула курса

```text
после каждого урока человек уходит не с "я узнал про AI",
а с новым действием в своем рабочем таймлайне
```

Если урок не меняет таймлайн, это не урок. Это лекционный шум. А шум у редакций и так есть.
