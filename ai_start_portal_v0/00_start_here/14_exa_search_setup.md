# Exa / search setup

Статус: рабочая памятка для курса  
Кому: спикеру, owners, продвинутым участникам  
Когда показывать: Deck 01, блок `Exa / search`

## Зачем

Exa нужен не как "еще один поисковик", а как способ дать AI нормальный входной слой: источники, ссылки, содержимое страниц, спорные места и свежий контекст.

На первом занятии цель простая: показать логику `search plan -> sources -> source map -> human gate`. Глубокую настройку API и агентов не делаем в общем темпе, чтобы не утонуть в технике.

## Где взять

- сайт: https://exa.ai/
- документация: https://exa.ai/docs/reference/getting-started
- dashboard/API key: https://dashboard.exa.ai/

## Вариант без кода

1. Открыть Exa или другой поисковый инструмент.
2. Взять 2-3 запроса из research prompt.
3. Найти три типа источников:
   - сильный;
   - слабый;
   - спорный.
4. Вернуть ссылки в ChatGPT / Claude.
5. Собрать source map и human gate.

## Вариант для owners через Codex / Claude Code

1. Создать в рабочей папке файл `search_prompt.md`.
2. Попросить агента собрать search plan и критерии источников.
3. Получить API key Exa в dashboard.
4. Положить ключ в локальную переменную окружения `EXA_API_KEY`.
5. Попросить агента собрать маленькую команду:

```text
Создай минимальный search tool для Exa.

Input:
- тема;
- 5 поисковых углов;
- критерии сильного / слабого источника.

Output:
- markdown-таблица source map;
- список слабых источников;
- human gate;
- что сохранить в hub.

Не пиши финальную публикацию.
Не добавляй фактов без источника.
```

## Что говорим участникам

Мы не просим вас завтра стать разработчиками. Мы показываем, что AI можно использовать не только как чат, но и как сборщика маленьких рабочих инструментов. Сегодня достаточно понять маршрут и сделать один безопасный search-прогон.
