# Purchase phrases for Ira

Статус: внутренние формулировки  
Назначение: коротко объяснить партнеру / клиенту, что докупаем и почему.

## Главная фраза

```text
Мы не покупаем набор нейросетей "на всякий случай".
Мы покупаем рабочий минимум для практики:
общий LLM, source layer, транскрипцию и 2-3 пилота для видео / визуала.
```

## Если спрашивают, почему не покупать все сразу

```text
До первых практик мы не знаем, какие инструменты реально встроятся в pipeline редакции.
Поэтому сначала покупаем базовый контур и пилоты.
Расширяем только то, что дает evidence: ускорение, качество, повторяемость.
```

## Если спрашивают, зачем ChatGPT / Claude / Codex

```text
ChatGPT или Claude нужен не как "чат для текстов", а как рабочий AI-слой.
Codex / Claude Code нужен owners и методологу, чтобы работать с hub:
структурой курса, markdown, промптами, gates, Q&A и обновлениями.
Участникам не обязательно всем идти в code-agent route.
```

## Если спрашивают, зачем Higgsfield

```text
Higgsfield берем не всем, а пилотно.
Он нужен visual / video группе для prompt projects, визуальных гипотез,
обложек, motion seeds и демонстрации visual conveyor.
После теста решаем, расширять или нет.
```

## Если спрашивают про OpusClip / Gling / Submagic

```text
Это не замена монтажеру.
Это acceleration layer: captions, rough cut, silence removal, кандидаты на клипы.
Берем 1-2 pilot seats, тестируем на безопасном видео и выбираем один рабочий route.
```

## Если спрашивают, что точно должно быть к старту

```text
К старту должны быть:
1. рабочий LLM для участников или пар;
2. доступ к hub;
3. safe source для практики;
4. transcript route;
5. правила, какие материалы можно загружать;
6. один visual/video pilot;
7. owner, который после занятия публикует запись, summary, Q&A и homework.
```

## Если надо сказать совсем коротко

```text
Покупаем не "нейросети", а контур практики.
Все, что не встроилось в работу после первых уроков, не расширяем.
```
