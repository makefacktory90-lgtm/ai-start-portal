# Прочитай сначала

Это рабочая папка AI Start.

Ваша задача не стать "AI-специалистом". Ваша задача - научиться брать обычный материал и получать черновик, который можно проверить и доработать.

## Самый простой путь

1. Откройте `02_workflows/01_note_to_news_draft/workflow.md`.
2. Скопируйте стартовую команду.
3. Вставьте свой материал.
4. Получите:
   - факты;
   - вопросы на проверку;
   - черновик новости;
   - Telegram-версию;
   - риски.
5. Проверьте результат по `03_quality_gates/factual_check.md`.
6. Сохраните пример или проблему.

## Что важно

- AI может ошибаться уверенно.
- Если факта нет в материале, его нельзя придумывать.
- Если сомневаетесь, пишите `нужно проверить`.
- Первый результат не обязан быть идеальным.
- Хорошие и плохие примеры нужны команде, чтобы улучшать workflows.

## Если сложно

Делайте только A1:

```text
готовый workflow -> вставить материал -> получить черновик -> проверить факты -> сохранить пример
```

Этого достаточно для старта.

## Три уровня погружения

Вы не обязаны проходить всё на максимальном уровне.

### A1: использовать

Ваша задача:

- взять готовый workflow;
- вставить материал;
- получить черновик;
- проверить факты;
- сохранить пример.

Это основной уровень для старта.

### A2: адаптировать

Ваша задача:

- взять результат A1;
- сделать версию для другого канала;
- сравнить варианты;
- отметить, где AI ошибся.

### A3: улучшить систему

Ваша задача:

- найти повторяющуюся проблему;
- описать evidence;
- предложить update;
- передать process owner.

Если у вас дедлайн и вы не хотите становиться AI-специалистом, делайте A1. Это нормальный рабочий минимум: готовый workflow, проверка результата и сохранение примера.
