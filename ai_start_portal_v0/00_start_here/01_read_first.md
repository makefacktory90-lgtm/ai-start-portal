# Прочитай сначала

Это рабочая папка AI Start.

Твоя задача не стать "AI-специалистом". Твоя задача - научиться брать обычный материал и получать черновик, который можно проверить и доработать.

## Самый простой путь

1. Открой `02_workflows/01_note_to_news_draft/workflow.md`.
2. Скопируй стартовую команду.
3. Вставь свой материал.
4. Получи:
   - факты;
   - вопросы на проверку;
   - черновик новости;
   - Telegram-версию;
   - риски.
5. Проверь результат по `03_quality_gates/factual_check.md`.
6. Сохрани пример или проблему.

## Что важно

- AI может ошибаться уверенно.
- Если факта нет в материале, его нельзя придумывать.
- Если сомневаешься, пиши `нужно проверить`.
- Первый результат не обязан быть идеальным.
- Хорошие и плохие примеры нужны команде, чтобы улучшать workflows.

## Если сложно

Делай только A1:

```text
готовый workflow -> вставить материал -> получить черновик -> проверить факты -> сохранить пример
```

Этого достаточно для старта.

## Три уровня погружения

Ты не обязан проходить всё на максималках.

### A1: использовать

Твоя задача:

- взять готовый workflow;
- вставить материал;
- получить черновик;
- проверить факты;
- сохранить пример.

Это основной уровень для старта.

### A2: адаптировать

Твоя задача:

- взять результат A1;
- сделать версию для другого канала;
- сравнить варианты;
- отметить, где AI ошибся.

### A3: улучшить систему

Твоя задача:

- найти повторяющуюся проблему;
- описать evidence;
- предложить update;
- передать process owner.

Если у тебя дедлайн и ты не хочешь становиться AI-специалистом, делай A1. Это нормальный рабочий минимум: готовый workflow, проверка результата и сохранение примера.
