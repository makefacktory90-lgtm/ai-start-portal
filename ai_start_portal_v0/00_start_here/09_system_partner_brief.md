# System partner brief

Статус: handoff v0.1  
Кому: системный партнер / практик  
Задача: не оценить красоту курса, а усилить внедрение.

## Контекст

AI Start - практический курс для телевизионных / редакционных команд.

Мы не учим людей "быть креативными с AI". Мы встраиваем AI в существующие рабочие маршруты:

1. Текст.
2. Видео.
3. Визуал.
4. Аудио.
5. Редакторский контур.

Главная логика:

```text
source
-> AI draft / AI assist
-> human gate
-> hub evidence
-> weekly update
```

## Что уже есть

- портал курса;
- 10 web-decks;
- 5 workflow pages;
- lesson contracts;
- participant prework;
- recordings / transcript / homework structure;
- prompt library;
- quality gates;
- client calibration notes;
- implementation ladder.

## Главный риск

У редакций нет свободного времени на "домашки".

Если участники не начнут делать маленькие внедрения после каждого урока, на финале будет нечего показывать.

Поэтому нужна система минимальных обязательных действий:

```text
20-40 минут после урока
один материал
один AI-шаг
один gate
один evidence
```

## Что нужно от партнера

### 1. Проверить implementation ladder

Документ:

- `00_start_here/08_implementation_ladder.md`

Вопросы:

- где action слишком большой;
- где action слишком слабый;
- где участники точно не сделают;
- где нужен owner;
- где можно встроить в уже существующую планерку / выпуск / монтаж / SMM-процесс.

### 2. Спроектировать Q&A chat

Нам нужен чат, который не распадается в хаос.

Проверить:

- какие каналы / теги нужны;
- какой SLA ответа реалистичен;
- кто отвечает;
- какие вопросы идут в Q&A;
- какие вопросы становятся updates в hub;
- как не давать участникам сливать конфиденциальные материалы.

### 3. Усилить evidence-модель

К финалу нужно показать не "мы провели 10 встреч", а:

- реальные outputs;
- approved examples;
- rejected examples;
- prompt updates;
- gate updates;
- time saved / friction notes;
- next automation candidates.

Вопрос:

```text
какой минимальный dashboard нужен, чтобы это было видно без бюрократии?
```

### 4. Проверить owner-модель

Нужно понять, кто внутри редакции держит:

- text workflow;
- video workflow;
- visual workflow;
- audio workflow;
- editorial contour;
- hub;
- security / policy.

Вопрос:

```text
какие owner roles обязательны, а какие можно не вводить, чтобы не раздуть систему?
```

### 5. Проверить финальный результат курса

Финал должен быть не презентацией, а рабочим kit:

```text
5 workflows
prompt library
quality gates
approved / rejected examples
owner map
recordings / Q&A
30-day adoption plan
automation candidates
```

Вопрос:

```text
чего не хватает, чтобы клиент реально продолжил после курса?
```

## Что НЕ нужно от партнера

Пока не нужны:

- еще 40 инструментов;
- абстрактная стратегия AI-трансформации;
- советы "надо больше вовлеченности";
- общие слова про культуру изменений.

Нужно:

```text
где встроить действие
кто owner
сколько минут
какой evidence
какой риск
что увидим на финале
```
