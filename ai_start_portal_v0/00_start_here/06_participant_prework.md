# Participant prework

Статус: prework checklist v0.1  
Назначение: что участник должен открыть / подготовить до каждого занятия, чтобы практика началась сразу.

## Общее правило

На практику можно брать только материал, который разрешено использовать в выбранном AI-инструменте.

Перед загрузкой материала в внешний сервис участник должен ответить:

```text
Это public, internal, restricted или synthetic source?
Что нужно обезличить?
Можно ли вставлять это во внешний AI-сервис?
Кто отвечает за публикационную проверку?
```

Если есть сомнение, используем synthetic source или обезличенный фрагмент.

## Lesson 01. Start map

Подготовить:

- одну повторяющуюся рабочую задачу;
- один пример, где AI мог бы ускорить первый проход;
- один страх / возражение по AI.

Fallback:

- если нет задачи, взять synthetic source из `09_practice_materials`.

## Lesson 02. Text OS

Подготовить:

- один текст / заметку / расшифровку, которую можно безопасно разобрать;
- если есть запрет на реальные материалы, взять synthetic source;
- открыть factual gate и prompt library.

Нужно иметь:

- доступ к ChatGPT / Claude / Gemini или любому корпоративно разрешенному LLM;
- текстовый редактор / заметки.

## Lesson 03. Video acceleration

Подготовить:

- транскрипт длинного видео, интервью, сюжета или эфира;
- если транскрипта нет, подготовить краткое описание с таймкодами;
- список каналов, куда обычно идут клипы.

Нужно иметь:

- LLM для анализа транскрипта;
- если есть доступ: Opus Clip / Gling / Descript / Premiere / DaVinci / Edius;
- если доступа нет: работаем на transcript-only practice.

## Lesson 04. Visual conveyor

Подготовить:

- один материал, для которого нужен визуал;
- 2-3 визуальных референса редакции;
- пример карточки / обложки / рубрики, которую нужно повторять.

Нужно иметь:

- Canva / Figma / Photoshop / любой рабочий редактор;
- если есть доступ: Higgsfield / Kling / Runway / Flux / Stable Diffusion;
- если доступа нет: делаем visual brief и prompt project без генерации.

## Lesson 05. Audio OS

Подготовить:

- один текст, который нужно превратить в речь;
- пример живого разговорного стиля, если есть;
- ограничения по voice tools.

Нужно иметь:

- LLM;
- диктофон / заметки;
- если разрешено: Whisper / MacWhisper / Adobe Podcast / Audition / Descript / ElevenLabs.

## Lesson 06. Editorial contour

Подготовить:

- 10-20 сигналов: темы, рубрики, новости, вопросы аудитории, наблюдения;
- список приоритетных тем;
- текущие каналы публикации.

Нужно иметь:

- таблицу или markdown-файл;
- LLM для группировки и digest.

## Lesson 07. Hub build

Подготовить:

- один удачный AI-output;
- один слабый / опасный AI-output;
- один пример редакторской правки;
- вопрос: что из этого должно стать правилом?

Нужно иметь:

- доступ к hub;
- owner или кандидат на owner.

## Lesson 08. Agents

Подготовить:

- одну повторяющуюся задачу, которую хочется полуавтоматизировать;
- список инструментов, которыми реально пользуется редакция;
- ограничения по данным и публикации.

Нужно иметь:

- lesson contracts;
- список инструментов, которыми реально пользуется редакция;
- AI policy.

## Lesson 09. Practice clinic

Подготовить:

- реальный кейс;
- material source;
- AI output;
- что получилось плохо / долго / рискованно;
- что хочется изменить.

Нужно иметь:

- 1 case note на группу или роль.

## Lesson 10. Final kit

Подготовить:

- 3-5 лучших examples;
- 3-5 rejected examples;
- список prompts / gates, которые реально пригодились;
- список owners;
- один next bet для теста после курса.

Нужно иметь:

- final kit checklist;
- 30-day dashboard.

## Если у участника нет доступа к инструментам

Это не блокер.

Минимальная практика строится так:

```text
source
-> prompt в демонстрационном инструменте спикера
-> участник анализирует output
-> gate
-> hub update
```

Важнее научиться ставить задачу, проверять output и сохранять evidence. Инструмент можно заменить. Плохой brief нельзя заменить ничем, кроме нормального brief. Увы, но тут физика.
