# Уровни выхода после занятия

Это не оценки и не соревнование.

Это ответ на вопрос: **что участник сможет делать иначе после лекции**.

Редакция состоит не из студентов. Люди уже умеют писать, монтировать, выпускать, проверять и принимать решения. Поэтому курс не должен продавать "магические тулзы". Он должен показать, где AI реально меняет таймлайн работы.

## Base: забирают все

Минимальный рабочий результат занятия.

Человек должен уметь:

- поставить задачу AI не как "сделай красиво", а как рабочий бриф;
- отделить source, задачу, формат, ограничения и human gate;
- получить проверяемый черновой output;
- понять, где AI помогает, а где начинается ответственность редактора;
- сохранить хороший / плохой пример в hub.

Результат:

```text
задача
-> source / материал
-> AI draft
-> human check
-> сохраненный пример
```

## Opti: ускорение своего рабочего места

Уровень для тех, кто хочет не просто попробовать, а ускорить конкретный кусок своей работы.

Человек должен уметь:

- адаптировать output под роль: корреспондент, редактор, монтажер, SMM, дизайнер, выпускающий;
- собрать 2-3 версии под разные каналы;
- понять, что быстрее руками, а что быстрее через AI;
- выбрать инструмент под задачу, а не задачу под инструмент;
- оформить маленький личный pipeline.

Результат:

```text
один материал
-> 2-3 рабочих output
-> сравнение с ручным способом
-> личный shortcut
```

## Advanced: собрать свой конвейер

Уровень для сильных практиков, process owners и будущих AI champions.

Человек должен уметь:

- собрать связку инструментов под повторяющуюся задачу;
- описать входы, выходы, проверки и владельца;
- улучшать prompt / context / gate по evidence;
- не автоматизировать хаос;
- отличить "прикольную демку" от производственного маршрута.

Результат:

```text
повторяющаяся задача
-> pipeline
-> tool stack
-> quality gate
-> update proposal
```

## Scout: куда смотреть дальше

Это не обязательная часть занятия, а окно в рынок.

Человек должен увидеть:

- какие инструменты уже меняют работу редакций;
- что пока сыро, но быстро улучшается;
- что можно тестировать на безопасных материалах;
- какие профессии в редакции усилятся первыми;
- какие навыки надо развивать, чтобы не отдать AI вкус, смысл и ответственность.

Результат:

```text
tool radar
-> что попробовать
-> где риск
-> кто в редакции может тестировать
```

## Главное правило

```text
Base = все уходят с новым действием
Opti = сильные ускоряют свой участок
Advanced = owners собирают систему
Scout = команда понимает, куда движется рынок
```

Если после занятия человек не может назвать, какой кусок его работы изменится завтра, занятие было слишком абстрактным.
