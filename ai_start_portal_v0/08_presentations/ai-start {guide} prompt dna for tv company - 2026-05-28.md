# Prompt DNA для ТВ-команды

## Назначение

Это короткая база по промптингу для редакции, маркетинга, HR и аналитических ролей.

Базовый принцип:

```text
хороший промпт = не магическая фраза,
а короткое рабочее задание с ясной структурой
```

## 1. ДНК промпта

Минимальная формула:

1. `Роль`
2. `Задача`
3. `Условия`
4. `Ограничения`

### 1. Роль

Кто именно должен помочь:

- редактор;
- аналитик;
- маркетолог;
- HR;
- ресерчер;
- продюсер;
- фактчекер;
- редактор Telegram;
- автор сценария;
- visual editor.

### 2. Задача

Что именно нужно сделать.

Примеры:

- собрать weekly digest;
- подготовить 5 тем для HR-контента;
- сравнить 4 вакансии;
- предложить структуру карточек;
- сделать spoken script;
- выделить темы из мониторинга;
- собрать campaign brief.

### 3. Условия

Что система получает на вход:

- заметку;
- расшифровку;
- список тем;
- вакансию;
- бриф;
- таблицу;
- ссылки;
- примеры хороших материалов;
- описания аудитории;
- ограничения канала.

### 4. Ограничения

Что нужно зафиксировать заранее:

- нельзя выдумывать факты;
- если данных не хватает, это нужно помечать отдельно;
- итог должен быть в понятном формате;
- человек отдельно проверяет рискованные места;
- стиль и тон должны соответствовать роли и каналу.

## 2. Базовый шаблон

```text
Ты помогаешь мне как [роль].

Задача:
[что нужно сделать и зачем].

Входные данные:
[какие материалы я даю].

Сделай:
1. [основной результат]
2. [дополнительный результат]
3. [риски / пробелы / вопросы]

Формат ответа:
[таблица / список / memo / outline / draft].

Ограничения:
- не выдумывай факты;
- если данных не хватает, пометь это отдельно;
- не добавляй лишнюю теорию;
- явно выдели, что должен проверить человек.
```

## 3. Как усиливать промпт

После первой версии полезно добавлять усилители.

### Усилитель 1. Уточняющие вопросы

```text
Перед ответом задай 3 уточняющих вопроса, которые помогут улучшить качество результата.
Если можно продолжить без них, пометь допущения отдельно.
```

### Усилитель 2. Оценка качества

```text
Оцени свой ответ по шкале от 1 до 10.
Если оценка ниже 9, объясни, что мешает результату быть сильнее.
Потом улучши ответ.
```

### Усилитель 3. 5-10 способов усиления

```text
Предложи 5-10 способов усилить этот промпт или результат.
Раздели их на: качество, конкретика, формат, риск, полезность для команды.
```

### Усилитель 4. Временные эксперты

```text
Подключи 3 временных экспертов:
- редактор;
- фактчекер;
- [нужная роль].

Пусть каждый коротко скажет, что улучшить.
Потом собери итоговую версию.
```

### Усилитель 5. Шляпы мышления

Можно не называть методологию, а просто разбить работу на режимы:

- факты;
- риски;
- идеи;
- сомнения;
- решение;
- следующий шаг.

Шаблон:

```text
Разбери задачу по шести режимам:
1. факты;
2. возможности;
3. риски;
4. слабые места;
5. рабочее решение;
6. следующий шаг.
```

### Усилитель 6. 2-3 варианта структуры

```text
Дай 3 версии структуры ответа:
- консервативную;
- рабочую;
- более сильную / амбициозную.

Коротко объясни, в чем разница.
```

### Усилитель 7. Pre-mortem

```text
Представь, что этот результат не сработал.
Назови 5 вероятных причин провала заранее.
```

### Усилитель 8. Postmortem

```text
Представь, что материал уже вышел и оказался слабым.
Сделай postmortem:
- что было недоформулировано;
- где возникли риски;
- каких данных не хватало;
- что нужно изменить в промпте в следующий раз.
```

### Усилитель 9. Human gate

```text
Перед финальной версией выдели блок:
что обязательно должен проверить человек перед использованием результата.
```

### Усилитель 10. Перевод в workflow

```text
Если этот промпт будет использоваться регулярно, предложи:
- как назвать workflow;
- какие входы он принимает;
- какой результат выдает;
- что сохранять в hub после каждого прогона.
```

## 4. Что важно проговорить на лекции

Важная мысль для участников:

> Модель знает много формул, frameworks и подходов. Не нужно помнить их все наизусть. Нужно уметь вовремя попросить нужный режим мышления, нужную роль и нужный тип проверки.

## 5. Упражнение для лекции

Это можно делать голосом, а ведущий заполняет шаблон на доске.

### Шаблон на доске

- роль
- задача
- условия
- ограничения
- усилитель
- human gate

### Пример

Карточка:

`Нужно подготовить темы для HR-контента на июнь`

Группа голосом дополняет:

- кто помогает;
- какая задача;
- какие входные данные нужны;
- какой формат результата нужен;
- что нельзя выдумывать;
- какой усилитель подключаем;
- что должен проверить человек.

## 6. Примеры для ТВ-команды

### Редакция

```text
Ты помогаешь как редактор.

Задача:
Из недельного мониторинга собрать 5 тем для публикаций.

Вход:
заметки, ссылки, даты, источники.

Сделай:
1. сгруппируй темы;
2. оцени значимость;
3. выдели идеи для публикаций;
4. отдельно пометь сомнительные источники.

Формат:
таблица.

Ограничения:
- не выдумывай факты;
- все спорное вынеси в блок "нужно проверить".
```

### Маркетинг

```text
Ты помогаешь как маркетолог.

Задача:
Подготовить 5 тем для HR-бренда на июнь.

Вход:
описание аудитории, tone of voice, примеры прошлых постов, темы июня.

Сделай:
1. предложи 5 тем;
2. объясни, почему они подходят аудитории;
3. укажи риски и пробелы.

Формат:
таблица.
```

### HR

```text
Ты помогаешь как HR-редактор.

Задача:
Сравнить 4 вакансии и найти слабые формулировки.

Вход:
тексты вакансий.

Сделай:
1. краткий аудит;
2. общие проблемы;
3. более ясные формулировки;
4. что обязательно проверить человеком.

Формат:
таблица + вывод.
```
