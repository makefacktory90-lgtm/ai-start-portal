# Speaker notes: Deck 02 / Text OS

## Главная рамка

Text OS - это не урок "как написать текст в ChatGPT". Это урок о том, как редакция превращает материал в проверяемый publication package.

Главная цепочка:

```text
source -> EXO -> ToV / Style Graph -> Repurpose -> QA -> hub update
```

## Практика внутри урока

Внутри занятия четыре коротких цикла:

| Блок | Время | Что делает участник |
|---|---:|---|
| EXO | 10 мин | разбирает source до первого текста |
| Smell test | 8 мин | находит AI-гладкость и заменяет ее конкретикой |
| Repurpose | 12 мин | собирает site / Telegram / social / headlines |
| QA | 8 мин | решает, что править: prompt, context, Style Graph или gate |

## Что участник должен унести

- EXO-разбор материала.
- Publication package: site, Telegram, social, headlines.
- Один риск, который поймал human gate.
- Одно предложение для обновления hub.

## Что важно сказать

- Красивый output не равен готовому материалу.
- Если факта нет в source, AI не имеет права его уверенно дорисовать.
- Repurpose - это смена задачи канала, а не сокращение.
- Хороший текст после AI должен звучать как редакция, а не как универсальный пресс-релиз.

## Что взять из подхода Тони

Берем структуру слоев:

- data layer: EXO;
- skill layer: repurpose chain;
- metric/gate layer: factual / tone / risk / format;
- role layer: разные outputs для разных ролей;
- flywheel: approved/rejected examples обновляют систему.

Не берем personal OS.
