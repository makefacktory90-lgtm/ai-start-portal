# Speaker notes: Deck 02 / Text OS + Style Graph

## Задача занятия

Это не лекция "как попросить AI написать текст".

Задача: показать редакции управляемый маршрут, где AI не заменяет редактора, а помогает разобрать источник, собрать варианты под форматы и пройти human gate.

Главная формула:

```text
source -> EXO -> Style Graph -> deplasticizer -> publication package -> QA -> hub update
```

## Как открывать

Сказать:

> Сегодня мы работаем с текстом. Но не с позиции "сделай красиво". Это слишком слабый запрос. Так рождается пластиковый текст: гладкий, уверенный, пустой и похожий на любой другой.
>
> Мы будем учиться давать AI не роль автора, а роль редакционного инструмента. Сначала он извлекает факты. Потом сверяется с голосом редакции. Потом помогает собрать версии под разные каналы. И только после этого человек принимает решение, что можно выпускать.

## Практика 01: EXO

Время: 10 минут.

Цель: не писать текст, а разобрать источник.

Промпт:

```text
Разбери материал как редакционный источник.

Не пиши финальный текст.

1. Выпиши проверяемые факты.
2. Выпиши людей, организации, места, даты, числа.
3. Выпиши цитаты отдельно от пересказа.
4. Отметь, чего не хватает для публикации.
5. Отметь места, где нельзя усиливать смысл.
6. Составь список вопросов редактору.

Если чего-то нет в источнике, пиши: "нет в источнике".
```

Что сказать после:

> Смотрите, мы сразу меняем качество работы. AI больше не пытается красиво выкрутиться. Он обязан показать: что есть в источнике, чего нет, где риск и где нужен человек.

## Практика 02: Style Graph и анти-пластик

Время: 12 минут.

Цель: поймать места, где текст звучит как универсальный AI-output.

Промпт:

```text
Проверь текст на пластиковую AI-гладкость.

Найди:
1. Универсальные фразы, которые подходят к любому материалу.
2. Раздувание события.
3. Уверенные утверждения без источника.
4. Канцелярский или рекламный тон.
5. Симметричные абзацы, которые звучат одинаково.

Для каждого места предложи замену через:
факт / цитату / место / героя / число / конкретное последствие.

Не переписывай весь текст сразу. Сначала покажи проблемные места и риск.
```

Что подчеркнуть:

> "Сделай человечнее" - плохая команда. Она ничего не объясняет. Хорошая команда говорит, что именно чинить: дым, масштаб, доказательство, ритм, риск.

## Практика 03: Publication package

Время: 15 минут.

Цель: из одного source собрать несколько форматов без потери фактов.

Промпт:

```text
На основе EXO и Style Graph собери publication package.

Форматы:
1. Сайт: заголовок, лид, структура, что проверить.
2. Telegram: первый экран, главный факт, зачем читать.
3. Social: короткий анонс без кликбейта.
4. 5 заголовков с разным фокусом.

Правила:
- не добавляй факты, которых нет в EXO;
- не усиливай масштаб события;
- сохраняй Style Graph;
- все сомнительное вынеси в "нужно проверить".
```

Что сказать:

> Repurpose - это не "укороти". Это смена задачи формата. Сайт объясняет. Telegram дает быстрый вход. Social держит одну мысль. Заголовки проверяют фокус.

## Финальный gate

Промпт:

```text
Проверь publication package перед редактором.

Сделай таблицу:
1. Факт / утверждение.
2. Есть ли это в источнике?
3. Риск: факт, тон, контекст, юридический, репутационный.
4. Что должен проверить человек.
5. Что нужно убрать или переписать.

Не утверждай, что текст готов к публикации.
Финальное решение принимает редактор.
```

Финальная мысль:

> Хороший AI-процесс не заканчивается текстом. Он заканчивается обновлением системы: какой пример сработал, какой пример нельзя повторять, какое правило нужно добавить в prompt, Style Graph или human gate.

## Что участник уносит

- EXO-разбор одного материала.
- Один список анти-пластиковых признаков.
- Один publication package.
- Один QA gate.
- Один update proposal для базы знаний редакции.
