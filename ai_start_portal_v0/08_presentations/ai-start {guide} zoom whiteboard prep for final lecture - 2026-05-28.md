# Как заранее подготовить Zoom Whiteboard для финальной лекции

## Короткий ответ

Да, доски в Zoom Whiteboard можно подготовить заранее, не во время встречи.

По официальной документации Zoom, whiteboard можно создавать:

- из Zoom desktop app;
- из Zoom web portal;
- из Zoom Web App.

Источники:

- Zoom Whiteboard getting started: https://support.zoom.com/hc/en/article?id=zm_kb&sysparm_article=KB0059671
- Whiteboards in meetings: https://support.zoom.com/hc/en/article?id=zm_kb&sysparm_article=KB0058148
- Whiteboard templates in meetings: https://support.zoom.com/hc/en/article?id=zm_kb&sysparm_article=KB0058133
- Present Canvas: https://support.zoom.com/hc/en/article?id=zm_kb&sysparm_article=KB0083533

## Что это значит practically

Если у вас платный Zoom и Whiteboards включен, вы можете:

1. заранее создать доски;
2. заранее расставить блоки и карточки;
3. открыть их потом прямо в meeting;
4. показать в Present mode;
5. самой двигать элементы, пока участники отвечают голосом.

Если у вас Whiteboards не включен или интерфейс неудобен, тогда Miro остается нормальным fallback.

## Как создать доску заранее

### Вариант 1. Через Zoom desktop app

1. Откройте Zoom desktop app.
2. В верхнем меню нажмите `Whiteboards`.
3. Справа сверху нажмите `New`.
4. Создайте пустую доску.
5. Назовите ее сразу понятно, например:
   - `TV21 Final - Prompt DNA`
   - `TV21 Final - Priorities`
   - `TV21 Final - 3 Month Plan`

### Вариант 2. Через Zoom web portal

1. Войдите в Zoom в браузере.
2. В левом меню откройте `Whiteboards`.
3. Нажмите `+ New Whiteboard`.
4. Сразу подпишите доску.

## Как открыть заготовленную доску во время встречи

1. Начните встречу в Zoom.
2. Внизу нажмите `Whiteboards`.
3. Откройте `Existing Whiteboards`.
4. Выберите нужную доску.
5. Откройте ее в режиме показа.
6. Включите `Present mode`, чтобы участники видели ровно то, что показываете вы.

## Какой режим выбрать

Для вашей лекции оптимально:

- участники отвечают только голосом;
- редактирование у участников выключено;
- вы одна двигаете карточки и печатаете короткие ответы.

То есть вам нужен сценарий `host-driven board`, а не совместное редактирование.

## Какие 3 доски подготовить

## Доска 1. Prompt DNA

Сделайте 6 прямоугольников:

- `Роль`
- `Задача`
- `Условия`
- `Ограничения`
- `Усилитель`
- `Human Gate`

Ниже поставьте 2-3 кейса отдельными карточками:

- `Темы для HR-контента на июнь`
- `Weekly digest из мониторинга`
- `Сравнение 4 вакансий`

Как использовать:

- вы читаете кейс;
- группа голосом предлагает, как заполнить блоки;
- вы кратко заносите ответы в прямоугольники.

## Доска 2. Матрица приоритетов

Нарисуйте 4 квадранта:

- `Высокая польза / Низкая сложность`
- `Высокая польза / Высокая сложность`
- `Низкая польза / Низкая сложность`
- `Низкая польза / Высокая сложность`

Подготовьте карточки:

- weekly digest;
- HR-посты;
- аудит вакансий;
- visual brief;
- spoken script;
- competitor monitoring;
- поиск ЛПР;
- автоматический outbound.

Как использовать:

- вы читаете одну карточку;
- участники голосом решают, куда ее положить;
- вы двигаете карточку.

## Доска 3. Дорожная карта на 3 месяца

Сделайте 3 колонки:

- `Месяц 1`
- `Месяц 2`
- `Месяц 3`

И 4 строки:

- `Что тестируем`
- `Кто отвечает`
- `Что сохраняем в hub`
- `Что считаем успехом`

Как использовать:

- сначала заполняете только `Месяц 1` текстовыми кейсами;
- потом добавляете `Месяц 2` как расширение в visual / audio / repurposing;
- потом `Месяц 3` как review, анализ и решение о следующем шаге.

## Как оформить визуально, чтобы не мучиться

1. Не делайте сложный дизайн.
2. Один цвет на одну логику:
   - синий: структура;
   - желтый: карточки;
   - зеленый: решения;
   - красный: риски / исключения.
3. Пишите только короткие слова, не абзацы.
4. Делайте крупный шрифт.
5. На каждой доске оставляйте пустое место для итогов.

## Что проверить заранее

1. Кнопка `Whiteboards` видна в Zoom.
2. Доска открывается до встречи.
3. `Present mode` включается.
4. Участники по умолчанию не редактируют доску.
5. Вы умеете:
   - двигать карточки;
   - писать короткий текст;
   - быстро переключаться между тремя досками.

## Когда лучше уйти в Miro

Miro лучше выбрать, если:

- в Zoom Whiteboard нет нужной функции у вашего аккаунта;
- интерфейс у вас не открылся заранее;
- вам привычнее Miro и вы не хотите учиться новому интерфейсу перед лекцией.

Если Zoom Whiteboard у вас открывается нормально, для этой встречи его достаточно.
