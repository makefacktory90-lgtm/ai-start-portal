# Lecture / deck system with embedded exercises

## 0. Правило презентаций

Каждая презентация строится как занятие, а не как доклад:

```text
thesis
-> why it matters
-> workflow map
-> live demo
-> A1 practice
-> A2 adaptation
-> A3 system update
-> hub action
```

Визуальный принцип: IraAI black / white / red, плотная редакционная подача, журнальные метки, минимум декоративности. У референса Тони Жуковой берем ритм и плотность, не контент.

Стратегическая ось:

```text
у команды уже есть профессия и процессы
-> AI подключается как черновой слой
-> human gate остается у профессионалов
-> удачные приемы сохраняются в hub
-> лидеры изменений решают, что становится стандартом
```

Источник решения:

- `../../ai_start_kit/ai-start {strategy} expert os company os learning frame - 2026-05-05.md`

## Deck 01. 14 мая: старт и карта AI-системы

Роль занятия: мягко показать, что курс не заменяет профессиональную работу, а помогает подключить AI к уже существующим редакционным процессам.

Slide spine:

1. `// СТАРТ` AI как рабочий слой редакции.
2. `// ГДЕ ТЕРЯЕТСЯ ПОЛЬЗА` Разовый удачный чат трудно повторить.
3. `// НОВЫЙ РАБОЧИЙ СЛОЙ` Материал -> AI draft -> human gate -> saved example.
4. `// РОЛИ` У каждой роли своя точка подключения AI.
5. `// ОБЩАЯ ПАМЯТЬ` Команда сохраняет workflows, prompts, gates и examples.
6. `// OS КОМПАНИИ` Лидеры изменений решают, что становится стандартом.
7. `// HUB` Место, где удачные приемы не теряются.
8. `// 5 МАРШРУТОВ` Пять пайплайнов как рабочие контуры, а не витрина инструментов.
9. `// HELICOPTER VIEW` Для каждого: input / AI step / human gate / output / hub artifact.
10. `// ZOOM-IN: TEXT OS` Source -> EXA -> context -> ToV -> Style Graph -> Repurpose Boss -> QA -> hub update.
11. `// A1 / A2 / A3` Использовать, адаптировать, предлагать улучшения.
12. `// HUMAN OWNS` AI drafts, human owns.
13. `// ПРАКТИКА` Embedded A1/A2/A3.
14. `// ДО 15 МАЯ` Что открыть и что принести.

Embedded exercise:

- A1: описать 5 pipeline pages через input / AI step / human gate / output.
- A2: выбрать один pipeline под свою роль и описать личную настройку OS для этого типа задачи.
- A3: предложить, что из личной практики должно попасть в общий hub редакции.

Hub action:

- сохранить role focus;
- отметить первый реальный материал-кандидат.

## Deck 02. 15 мая: Text OS / материал -> текстовые форматы

Роль занятия: показать, как один исходник превращается не просто в текст, а в управляемую систему текстовых форматов.

Slide spine:

1. `// НЕ ПРОМПТ, А TEXT OS` Один исходник не равен одному формату.
2. `// EXA` Facts, people, dates, quotes, meanings, gaps, risks.
3. `// CONTEXT` Что AI должен знать перед письмом.
4. `// TONE OF VOICE` Как не потерять голос редакции / рубрики / ведущего.
5. `// STYLE GRAPH` Ритм, словарь, запреты, доказательства, типовые конструкции.
6. `// REPURPOSE BOSS` Site / Telegram / social / captions / cards / talking points.
7. `// QA` Facts, tone, legal/reputation, channel risk.
8. `// LIVE DEMO` Material 01 -> Text OS starter prompt.
9. `// ПРАКТИКА A1` Собрать безопасный пакет.
10. `// ПРАКТИКА A2` Собрать ToV / Style Graph draft и пересобрать форматы.
11. `// ПРАКТИКА A3` Сделать update proposal или skill proposal.
12. `// HUB ACTION` Сохранить good / risky examples и обновление Text OS.

Embedded exercise:

- A1: EXA, check, site, Telegram, social, headlines, risks.
- A2: ToV / Style Graph draft, Repurpose Boss map.
- A3: обновить prompt, factual gate или предложить skill для hub.

Hub action:

- сохранить один хороший output;
- сохранить один слабый output как warning example.
- сохранить один update proposal для Text OS.

## Deck 03. 19 мая: длинное видео -> вертикальные клипы

Роль занятия: показать не сервисы, а производственный маршрут для коротких видео.

Slide spine:

1. `// НЕ ОБЗОР СЕРВИСОВ` Нам нужен маршрут, а не витрина инструментов.
2. `// LONG VIDEO` Что можно и нельзя нарезать.
3. `// TRANSCRIPT FIRST` Если AI не видит видео, работаем по тексту и таймкодам.
4. `// CLIP CANDIDATES` 5-7 кандидатов, не финальное решение.
5. `// TOP-3` Смысл, hook, risk, монтажная пригодность.
6. `// CAPTIONS` Subtitle chunks как отдельный слой.
7. `// COVER / TITLE` Текст остается редактируемым.
8. `// LIVE DEMO` Material 02 -> candidates.
9. `// ПРАКТИКА` A1/A2/A3.
10. `// HUB ACTION` Сохранить таблицу кандидатов и video gate update.

Embedded exercise:

- A1: выбрать 5-7 кандидатов и топ-3.
- A2: собрать publication package.
- A3: обновить video gate.

Hub action:

- сохранить таблицу кандидатов;
- сохранить правила против кликбейта и потери контекста.

## Deck 04. 19 мая: материал -> визуал и карточки

Роль занятия: отделить визуальное мышление от "нейросеть сделай картинку".

Slide spine:

1. `// ВИЗУАЛ НЕ КАРТИНКА` Визуал = смысл + формат + слой текста + gate.
2. `// FACTS STAY EDITABLE` Факты не запекаем в image model.
3. `// FORMATS` Cover, title, intertitle, cards, quote, interactive quad.
4. `// VISUAL BRIEF` Что должен получить дизайнер / шаблон / монтажер.
5. `// AI ASSET` AI как фон, метафора, вариант, не источник фактов.
6. `// VISUAL BUILDER` Учебный prototype принципа.
7. `// LIVE DEMO` Material 03 -> visual brief.
8. `// ПРАКТИКА A1` Структура карточек.
9. `// ПРАКТИКА A2/A3` Brief + visual-builder check.
10. `// HUB ACTION` Сохранить brief и calibration slots.

Embedded exercise:

- A1: главный смысл, факты, 3 формата, структура карточек.
- A2: visual brief.
- A3: собрать учебный prototype и отметить, что заменим после 10 мая.

Hub action:

- сохранить visual brief;
- сохранить список реальных форматов, которые извлечем из клиентских материалов.
