## Когда использовать

Используйте, если есть:

- заметка с события;
- расшифровка видео / интервью / эфира;
- пресс-релиз;
- готовый ТВ-сюжет;
- набор фактов от коллеги;
- архивный материал, который нужно переупаковать.

## Рабочая схема Text OS

```text
source
-> EXO
-> context
-> Tone of Voice
-> Style Graph
-> Repurpose Boss
-> formats
-> QA
-> skill / workflow update
```

Text OS не равен "сделайте новость". Это редакционный контур, где AI сначала помогает разобрать материал, затем собрать версии под разные каналы, а человек держит факты, тон, риски и решение о выпуске.

### EXO

EXO - слой извлечения из материала:

- факты;
- действующие лица / организации / места;
- даты, числа, должности;
- цитаты и кто их сказал;
- смысловые тезисы;
- что утверждается без доказательства;
- что нужно проверить;
- чего не хватает для публикации;
- юридические, репутационные и редакционные риски.

### Tone of Voice

Tone of Voice задает, как материал звучит в конкретной редакции, рубрике или у конкретного ведущего:

- уровень официальности;
- допустимая эмоциональность;
- словарь;
- длина фраз;
- отношение к иронии, оценочным словам и драматизации;
- запретные конструкции.

Если ToV еще нет, участник делает временную версию на основе 3-5 примеров и помечает ее как `draft`.

### Style Graph

Style Graph - карта стиля, а не красивый брендбук. В ней фиксируются:

- ритм текста;
- типовые заходы;
- слова, которые редакция использует;
- слова, которые редакция не использует;
- как доказываются тезисы;
- как оформляются цитаты, списки, цифры;
- какие ошибки повторяются.

Style Graph нужен, чтобы разные участники не получали пять несовместимых голосов из одного материала.

### Repurpose Boss

Repurpose Boss - слой пересборки одного материала в разные форматы:

- сайт;
- Telegram;
- social-анонс;
- заголовки;
- карточки;
- подписи к видео;
- intro для аудио;
- тезисы для ведущего;
- internal brief для редактора.

Задача не "укоротить", а сменить цель формата и сохранить факты.

### Skill Agent

Skill Agent - контур, который превращает повторяемую удачную работу в reusable workflow:

- что сработало;
- на каком материале;
- какой prompt помог;
- какой gate поймал ошибку;
- что стоит добавить в общий hub;
- кто owner обновления.

Это A3-слой курса: участники не только используют pipeline, но и улучшают систему.

## Не использовать без редактора

- конфликтные темы;
- обвинения;
- происшествия с пострадавшими;
- юридически рискованные формулировки;
- материалы с персональными данными;
- любые утверждения, где нет понятного источника.

## Стартовая команда

```text
Открой AI Start Kit.
Используйте Pipeline 01: Text OS / материал -> текстовые форматы.

Задача:
1. Сначала сделайте EXO-разбор материала:
   - факты;
   - лица / организации / места;
   - даты, числа, должности;
   - цитаты;
   - смысловые тезисы;
   - что нужно проверить;
   - какого контекста не хватает;
   - редакционные, юридические и репутационные риски.
2. Определите задачу форматов:
   - сайт;
   - Telegram;
   - social;
   - заголовки;
   - дополнительные форматы, если они нужны редакции.
3. Примените Tone of Voice.
   Если ToV не передан, соберите временный ToV draft из исходного материала и явно пометьте его как draft.
4. Примените Style Graph.
   Если Style Graph не передан, предложите минимальный Style Graph draft: ритм, словарь, запреты, доказательства, типовые конструкции.
5. Сделайте карту пересборки через Repurpose Boss:
   - что сохраняется неизменным;
   - что меняется под канал;
   - где нельзя усиливать смысл;
   - какой формат нужно показать редактору первым.
6. Соберите publication package:
   - черновик для сайта;
   - Telegram-версию;
   - social-анонс;
   - 5 вариантов заголовка с разным фокусом;
   - при необходимости подписи / карточки / тезисы для ведущего.
7. Не выдумывайте факты, имена, цифры, цитаты, причины или источники.
8. Все сомнительное вынесите в блок "нужно проверить".
9. В конце дайте QA-gate и update proposal: что стоит сохранить в hub или улучшить в workflow.

Материал:
[вставьте материал ниже]
```

## A1: безопасный первый пакет

Сделайте базовый output:

```text
EXO -> нужно проверить -> сайт -> Telegram -> social -> заголовки -> риски
```

Цель A1: получить рабочий черновой пакет и не наплодить красивых несуществующих фактов.

## A2: адаптация под канал

После A1 используйте:

```text
Сравните версии для сайта, Telegram и social.
Покажите таблицу:
- цель формата;
- что меняется в подаче;
- какие факты нельзя искажать;
- где тон стал рискованным;
- какую версию лучше показать редактору первой.
```

Цель A2: понять, что репаковка - это не "укороти", а смена задачи формата.

## A2+: Style Graph / Repurpose Boss

Если у редакции уже есть примеры, используйте:

```text
На основе этих 3-5 примеров сделайте Style Graph draft:
- ритм;
- словарь;
- запреты;
- типовые заходы;
- как доказываются тезисы;
- где текст звучит не как наша редакция.

Затем пересоберите исходный материал через Repurpose Boss:
- сайт;
- Telegram;
- social;
- подписи / карточки / тезисы для ведущего.

Покажите, какие решения изменились из-за Style Graph.
```

## A3: улучшение workflow

Если AI ошибся, используйте:

```text
Сделайте update proposal для Pipeline 01.
Укажите:
- какая ошибка повторяется;
- пример из output;
- почему это риск;
- что нужно добавить в workflow или quality gate;
- как проверить изменение на 3 материалах.
```

Цель A3: превратить повторяющуюся правку редактора в улучшение общей системы.

## A3+: Skill Agent

Если найден удачный повторяемый прием, используйте:

```text
Сделайте skill / workflow proposal для AI-hub.

Опишите:
- какую задачу решает прием;
- для какой роли он подходит;
- какой input нужен;
- какие шаги повторяются;
- какой prompt или checklist нужен;
- какие примеры нужно сохранить;
- какие ошибки должен ловить gate;
- кто owner обновления.
```

## Quality gate

Перед показом редактору проверьте:

- все факты есть в исходном материале;
- цитаты не придуманы;
- причины и последствия не додуманы;
- должности, имена, даты и числа вынесены на проверку;
- заголовок не обещает больше, чем есть в материале;
- Telegram-версия не упрощает до искажения;
- social-анонс не превращает новость в рекламу;
- есть блок "нужно проверить".

## Что сохранять в hub

- хороший output;
- плохой output с заметкой, что именно сломалось;
- редакторскую правку, если она повторилась;
- новый вопрос, который надо добавить в quality gate.
