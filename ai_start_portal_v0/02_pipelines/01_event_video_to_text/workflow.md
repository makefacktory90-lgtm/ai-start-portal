# Pipeline 01: событие / видео -> текстовые форматы

Статус: starter v0.1  
Для блока: 15 мая, "Событие -> текстовые форматы"  
Подходит для: корреспондент, редактор, SMM, выпускающий  
Главный результат: один исходный материал превращается в проверяемый пакет для сайта, Telegram, соцсетей и заголовков.

## Когда использовать

Используй, если есть:

- заметка с события;
- расшифровка видео / интервью / эфира;
- пресс-релиз;
- готовый ТВ-сюжет;
- набор фактов от коллеги;
- архивный материал, который нужно переупаковать.

## Не использовать без редактора

- конфликтные темы;
- обвинения;
- происшествия с пострадавшими;
- юридически рискованные формулировки;
- материалы с персональными данными;
- любые утверждения, где нет понятного источника.

## Стартовая команда

```text
Открой AI Start Kit.
Используй Pipeline 01: событие / видео -> текстовые форматы.

Задача:
1. Сначала извлеки только факты из материала.
2. Отдельно выпиши, что нужно проверить.
3. Отдельно выпиши, какого контекста не хватает.
4. Сделай черновик новости / материала для сайта.
5. Сделай Telegram-версию.
6. Сделай короткий social-анонс.
7. Дай 5 вариантов заголовка с разным фокусом.
8. Не выдумывай факты, имена, цифры, цитаты, причины или источники.
9. Все сомнительное вынеси в блок "нужно проверить".
10. В конце дай риски для редактора.

Материал:
[вставь материал ниже]
```

## A1: безопасный первый пакет

Сделай только базовый output:

```text
факты -> нужно проверить -> сайт -> Telegram -> social -> заголовки -> риски
```

Цель A1: получить рабочий черновой пакет и не наплодить красивых несуществующих фактов.

## A2: адаптация под канал

После A1 попроси:

```text
Сравни версии для сайта, Telegram и social.
Покажи таблицу:
- цель формата;
- что меняется в подаче;
- какие факты нельзя искажать;
- где тон стал рискованным;
- какую версию лучше показать редактору первой.
```

Цель A2: понять, что репаковка - это не "укороти", а смена задачи формата.

## A3: улучшение workflow

Если AI ошибся, попроси:

```text
Сделай update proposal для Pipeline 01.
Укажи:
- какая ошибка повторяется;
- пример из output;
- почему это риск;
- что нужно добавить в workflow или quality gate;
- как проверить изменение на 3 материалах.
```

Цель A3: превратить повторяющуюся правку редактора в улучшение общей системы.

## Quality gate

Перед показом редактору проверь:

- все факты есть в исходном материале;
- цитаты не придуманы;
- причины и последствия не додуманы;
- должности, имена, даты и числа вынесены на проверку;
- заголовок не обещает больше, чем есть в материале;
- Telegram-версия не упрощает до искажения;
- social-анонс не превращает новость в рекламу;
- есть блок "нужно проверить".

## Что сохранять в hub

- хороший output;
- плохой output с заметкой, что именно сломалось;
- редакторскую правку, если она повторилась;
- новый вопрос, который надо добавить в quality gate.
