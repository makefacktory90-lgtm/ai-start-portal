## Когда использовать

Используй, если нужно сделать:

- аудиоверсию статьи;
- 60-90 секунд spoken script;
- короткий подкастовый фрагмент;
- intro для аудио;
- текст для озвучки;
- адаптацию материала из письменного в разговорный формат.

## Важное ограничение

AI не должен добавлять факты "для живости". Разговорность не равна выдумке.

## Стартовая команда

```text
Открой AI Start Kit.
Используй Pipeline 04: материал -> аудио.

Задача:
1. Извлеки главный смысл материала.
2. Выпиши факты, которые должны сохраниться точно.
3. Отдельно выпиши, что нужно проверить.
4. Сделай spoken script на 60-90 секунд.
5. Сделай короткий intro hook.
6. Сделай версию для подкастового фрагмента с естественными переходами.
7. Укажи места, где ведущему лучше говорить своими словами.
8. Не добавляй факты, эмоции или оценки, которых нет в материале.

Материал:
[вставьте ниже]
```

## A1: сделать spoken script

Сделай:

```text
смысл -> факты -> intro -> script 60-90 sec -> риски
```

## A2: адаптировать под голос

После A1 попроси:

```text
Сделай 2 версии:
1. нейтральная редакционная;
2. более разговорная для ведущего.

Потом сравни:
- где речь стала живее;
- где появился риск лишней оценки;
- какие факты нельзя менять;
- какие места ведущий должен проверить.
```

## A3: обновить audio gate

Если текст звучит искусственно, попроси:

```text
Сделай update proposal для audio workflow.
Укажи:
- какие фразы звучат как AI;
- где потерян человеческий ритм;
- где появился риск фактической ошибки;
- какое правило добавить;
- как проверить на 3 аудио-скриптах.
```

## Audio quality gate

Проверить:

- текст можно произнести вслух без спотыканий;
- в первом абзаце понятно, зачем слушать;
- нет длинных письменных конструкций;
- нет придуманных эмоций;
- факты не исказились при разговорной адаптации;
- ведущий понимает, где нужны свои слова;
- script не звучит как пресс-релиз, прочитанный роботом.
