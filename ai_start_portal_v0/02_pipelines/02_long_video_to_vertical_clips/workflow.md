# Pipeline 02: длинное видео -> вертикальные клипы

Статус: starter v0.1  
Для блока: 19 мая, "Вертикальное видео и визуал"  
Подходит для: видео-специалист, SMM, редактор, выпускающий  
Главный результат: длинное видео или транскрипт превращается в 3-5 кандидатов на вертикальные клипы с хуками, субтитрами, обложками и публикационным пакетом.

## Когда использовать

Используй, если есть:

- длинный сюжет / репортаж 6-8 минут;
- фрагмент эфира;
- интервью;
- расшифровка видео;
- готовый ТВ-сюжет;
- архивный выпуск, из которого надо выбрать короткие фрагменты.

## Важное ограничение

Если у AI нет доступа к самому видео, работаем по транскрипту, монтажному плану или ручным таймкодам.

AI не выбирает финальный фрагмент за редактора. AI предлагает кандидаты, человек проверяет смысл, контекст, права, качество и монтажную возможность.

## Стартовая команда

```text
Открой AI Start Kit.
Используй Pipeline 02: длинное видео -> вертикальные клипы.

Задача:
1. Проанализируй материал как источник для коротких вертикальных клипов.
2. Найди 5-7 потенциальных фрагментов.
3. Для каждого фрагмента укажи:
   - примерный таймкод или место в транскрипте;
   - главный смысл;
   - почему это может работать как короткий клип;
   - hook первой фразы;
   - текст на обложку;
   - описание для публикации;
   - риск потери контекста.
4. Выбери 3 лучших кандидата.
5. Для 3 лучших кандидатов сделай subtitle chunks короткими строками.
6. Не выдумывай цитаты и факты.
7. Все сомнительное вынеси в блок "нужно проверить".

Материал / транскрипт / монтажный план:
[вставь ниже]
```

## A1: выбрать кандидаты

Сделай:

```text
5-7 кандидатов -> топ-3 -> hook -> обложка -> описание -> риски
```

Цель A1: быстро увидеть, есть ли в длинном материале короткие фрагменты, которые не убивают смысл.

## A2: подготовить публикационный пакет

После A1 попроси:

```text
Для каждого из топ-3 клипов сделай полный публикационный пакет:
- hook до 8 слов;
- первые 3 секунды;
- subtitle chunks;
- текст обложки;
- описание;
- Telegram-анонс;
- что должен проверить редактор;
- что должен проверить монтажер.
```

## A3: обновить видео-gate

Если AI предлагает слабые или рискованные клипы, попроси:

```text
Сделай update proposal для video quality gate.
Укажи:
- какой тип плохих клипов AI предлагает;
- почему это риск;
- какие правила отбора добавить;
- какие 3 примера использовать для теста.
```

## Video quality gate

Проверить перед производством:

- фрагмент не теряет критичный контекст;
- hook не обещает больше, чем есть в видео;
- цитата не искажена;
- субтитры не ломают смысл;
- обложка не кликбейтит;
- есть права / разрешение на публикацию;
- качество картинки и звука пригодно;
- есть редакторское решение, зачем этот клип нужен.
