## Когда использовать

Используй для:

- утреннего мониторинга;
- weekly digest;
- идей для контент-сетки;
- регулярных Telegram-форматов;
- разбора архива;
- tone-of-voice калибровки;
- еженедельного update log;
- минимальной аналитики.

## Главное ограничение

Редакторский контур не должен превращаться в автопубликацию.

AI группирует, предлагает и подсвечивает риски. Редактор решает, что важно, что публиковать и что требует проверки.

## Стартовая команда: monitoring -> digest

```text
Открой AI Start Kit.
Используй Pipeline 05: редакторский контур.

Задача:
1. Прочитай строки мониторинга.
2. Сгруппируй сигналы по темам.
3. Отметь важность каждой темы.
4. Отдельно выпиши источники, которым нельзя доверять без проверки.
5. Предложи идеи материалов.
6. Предложи, что может пойти в Telegram.
7. Не выдавай сомнительные сигналы за факты.
8. Все спорное вынеси в "нужно проверить".

Строки мониторинга:
[вставьте ниже]
```

## A1: получить digest

Сделай:

```text
строки мониторинга -> темы -> важность -> идеи -> риски источников
```

## A2: собрать неделю

После A1 попроси:

```text
На основе digest предложи контент-сетку на неделю:
- сайт;
- Telegram;
- соцсети;
- видео;
- визуал;
- аудио.

Для каждой позиции укажи пайплайн и что нужно проверить.
```

## A3: обновить AI-hub

Если повторяются проблемы, попроси:

```text
Сделай weekly update proposal для AI-hub.
Укажи:
- какие 3 проблемы повторились;
- evidence;
- какой файл надо обновить: workflow, context, quality gate, prompt library или examples;
- proposed change;
- owner;
- статус: proposed / testing / approved / rejected / personal.
```

## Editorial contour quality gate

Проверить:

- источник известен;
- сомнительный источник помечен;
- идея материала не выдана за факт;
- контент-сетка реалистична;
- есть owner или next step;
- Telegram-регулярность не заменяет редакционный смысл;
- weekly update основан на evidence, а не на ощущении "что-то не то".
