# Deck 01 safe source practice

Статус: учебный материал для первого занятия  
Назначение: общий безопасный source для практики `research prompt -> источники -> Source Map -> human gate -> share`.

Материал синтетический. Его можно использовать в AI-сервисах на занятии: в нем нет персональных данных, коммерческой тайны, внутренних документов клиента и непубличных деталей.

## Source

В городе открыли обновленную общественную мастерскую для школьников и студентов.

Площадка работает на базе городского молодежного центра. В ней есть зона для 3D-печати, небольшой медиакласс, рабочие столы для командных проектов и открытые консультации с наставниками.

По словам координатора проекта, мастерская должна помочь подросткам и студентам пробовать инженерные, медийные и исследовательские проекты без покупки дорогого оборудования.

В первый месяц команда планирует провести вводные занятия по 3D-моделированию, созданию коротких видео, работе с открытыми данными и подготовке проектных заявок.

Участие в занятиях бесплатное, но нужна предварительная регистрация. Организаторы отдельно отмечают, что мастерская не заменяет школьные и вузовские программы, а должна стать местом для практических проб и командной работы.

Ожидается, что первые проекты участников покажут на открытом дне в конце месяца.

## Что можно сказать уверенно

- открылась обновленная общественная мастерская;
- площадка работает на базе молодежного центра;
- есть 3D-печать, медиакласс, рабочие места и наставники;
- занятия бесплатные, нужна регистрация;
- первые проекты хотят показать на открытом дне в конце месяца.

## Что нельзя додумывать

- точный адрес, если он не дан;
- имена участников;
- бюджет проекта;
- сколько человек уже записалось;
- что мастерская точно "изменит образование в городе";
- что оборудование доступно без ограничений.

## Практика 30 минут

### 0. Research prompt

Промпт вдохновлен multi-agent research подходом Тони Жуковой: сначала ищем с разных углов, потом критикуем источники, потом превращаем находки в редакционную карту.

Референс:

- https://github.com/tonyazhuuki/deep-research-skill
- https://github.com/tonyazhuuki/tldr_buddy

```text
Ты research-команда для редакции ТВ / digital.

Задача: помочь подготовить материал по source ниже.
Не пиши публикацию. Не делай финальный текст.
Сначала сделай проверяемую исследовательскую карту.

Роли:
1. SCOUT FACTS: ищет факты, даты, действующих лиц, место, первоисточники.
2. SCOUT LOCAL: ищет локальный / городской / отраслевой контекст.
3. SCOUT SKEPTIC: ищет, что может быть преувеличено, спорно или не доказано.
4. SCOUT FRESH: проверяет, что могло измениться недавно.
5. CRITIC: отсекает слабые, рекламные, непроверяемые источники.
6. ACTION MAPPER: переводит находки в редакционные действия.

Сделай:
1. Search plan: 8-10 запросов, сгруппированных по ролям.
2. Для каждого запроса: что ищем, какой источник будет сильным, какой будет слабым.
3. Source map:
   - source;
   - link;
   - что подтверждает;
   - что не подтверждает;
   - риск;
   - можно ли использовать;
   - что проверить руками.
4. Unknowns tracker: чего не хватает для уверенной публикации.
5. Anti-plastic check: какие формулировки будут звучать как AI-клише.
6. Action map:
   - что можно сделать для сайта;
   - что можно сделать для Telegram;
   - что можно сделать для короткого social-анонса;
   - что пока нельзя выпускать.
7. Human gate: список проверок перед публикацией.

Правила:
- не додумывай факты;
- не подменяй источник красивой формулировкой;
- отделяй "можно сказать" от "хочется сказать";
- если источник слабый, прямо скажи почему;
- если данные нужны свежие, пометь это отдельно;
- не используй внутренние или персональные данные.
Если данных нет, пиши: "нет в источнике".
```

### Как использовать Exa / search

Exa: https://exa.ai/

Памятка по подключению: [Exa / search setup](../00_start_here/14_exa_search_setup.md)

Список skills и ссылок по урокам: [lesson skill stack](../00_start_here/15_lesson_skill_stack.md)

Вариант без кода:

```text
1. Открыть Exa / Perplexity / Tavily / обычный поиск.
2. Взять 3-5 запросов из search plan.
3. Сохранить 3 источника: сильный, слабый, спорный.
4. Вернуться в LLM и собрать Source Map.
5. Отдельно выписать human gate.
```

```text

3. Установить / изучить deep-research-skill:
   https://github.com/tonyazhuuki/deep-research-skill
4. Если есть API-ключ Exa, подключить search endpoint позже как отдельный tool.
5. Сохранить удачный prompt в hub.
```

На первом занятии достаточно понять логику. Глубокую настройку Exa / API / agents делаем позже.

### 1. Source Map

Промпт:

```text
Разбери материал как редакционный source.

Не пиши финальную публикацию.

Собери таблицу:
1. Проверяемый факт из источника.
2. Где это видно в source.
3. Что можно сказать уверенно.
4. Что нельзя додумывать.
5. Какой формат подходит: сайт, Telegram, social, карточки.
6. Риск перед публикацией.
7. Что должен проверить человек.

Если данных нет, пиши: "нет в источнике".
```

### 2. Smell test

Промпт:

```text
Проверь черновик на AI-гладкость.

Ищи:
- фразы, которые подходят к любому похожему материалу;
- раздувание масштаба;
- канцелярит;
- рекламный тон;
- уверенные выводы без фактов;
- одинаковый ритм предложений.

Сначала покажи проблемные фразы.
Потом предложи замену через факт, место, действие, цитату или ограничение.
Не добавляй новых фактов.
```

### 3. Human gate

Участник формулирует одну проверку:

```text
Перед выпуском человек обязан проверить:
...
```

Примеры:

- адрес и расписание;
- кто организатор;
- где регистрация;
- есть ли ограничения по возрасту;
- можно ли использовать формулировку "бесплатно" без условий.

### 4. Share

В чат / общий документ отправить:

```text
1. Один факт из Source Map.
2. Один AI-smell.
3. Один human gate.
4. Что из этого стоит сохранить в hub.
```
