# Упражнения по всем встречам

## Как читать

У каждого дня есть три уровня:

- A1: участник проходит готовый workflow и получает рабочий результат;
- A2: участник адаптирует output под реальный формат / роль / канал;
- A3: участник улучшает систему: prompt, gate, example или hub.

Это не домашка ради домашки. Каждое упражнение должно оставить след в AI-hub.

## 14 мая. Старт и карта AI-системы

Цель: понять, что курс строится не вокруг "нейросетей" и не вокруг "набора промптов", а вокруг новой рабочей системы.

Главная рамка:

```text
OS роли / специалиста
-> OS редакции
-> OS компании
```

A1:

```text
Открой hub.
Найди 5 pipeline pages.
Для каждого pipeline выпиши:
- какую редакционную задачу он решает;
- какой input нужен;
- какой output должен получиться;
- кто в редакции должен проверять результат.
```

A2:

```text
Выбери один pipeline, который нужен твоей роли чаще всего.
Опиши свою рабочую сборку для этого pipeline:
- ваша роль;
- тип материала;
- канал;
- где сейчас тратится время;
- какой output был бы полезен;
- где нужна человеческая проверка;
- что из этого должно быть личной настройкой, а что общим правилом редакции.
```

A3:

```text
Предложи один элемент, который должен попасть из личной практики в AI-hub редакции:
- зачем она нужна;
- кто owner;
- какие примеры туда сохранять;
- как понять, что страница полезна.
```

Hub action: сохранить role focus, кандидата на первый реальный материал и один элемент для OS редакции.
