# Как подключить Higgsfield MCP и генерировать изображения и видео через агента

Статус: участнический гайд  
Для блока: "Видео, визуал и агентные пайплайны"  
Подходит для: редактор, продюсер, SMM-специалист, видеоредактор, автор визуальных материалов  
Главный результат: Higgsfield подключен к Claude / Claude Code / Codex как внешний инструмент, а генерация изображений и коротких видео запускается из агентного чата.

## Назначение

Higgsfield MCP нужен, когда вы хотите работать с изображениями и видео не только через интерфейс сервиса, но и внутри агента.

Обычный сервисный сценарий выглядит так:

```text
зайти в сервис
-> вспомнить стиль
-> написать промпт
-> запустить генерацию
-> скачать результат
-> вернуться в рабочий проект
```

Агентный сценарий выглядит иначе:

```text
описать задачу в Codex / Claude
-> агент использует сохраненный стиль и правила
-> агент запускает Higgsfield
-> агент возвращает результат или ссылку
-> человек утверждает вариант
```

Важно: каждому пользователю нужен собственный MCP URL или MCP token из аккаунта Higgsfield. Не используйте общий токен преподавателя, коллеги или команды в публичных документах.

## Что нужно до старта

### 1. Проверьте аккаунт Higgsfield

1. Откройте [higgsfield.ai](https://higgsfield.ai/).
2. Войдите в свой аккаунт.
3. Проверьте, что у аккаунта есть кредиты или подходящий тариф.
4. Откройте страницу аккаунта: `Account`, `Profile`, `Settings`, `API`, `Integrations` или `Connectors`.
5. Найдите раздел, где создается MCP URL, MCP token или API key.

Названия пунктов интерфейса могут меняться. Смысл один: персональная ссылка или токен берутся внутри вашего аккаунта Higgsfield.

### 2. Выберите среду подключения

Можно использовать один из вариантов:

- **Claude web / Claude Desktop** - самый простой путь через `Settings -> Connectors`.
- **Claude Code** - подключение через команду `claude mcp add`.
- **Codex** - подключение через Codex CLI или конфигурацию MCP, если ваша версия это поддерживает.

Если вы впервые подключаете MCP, начните с Claude web или Claude Desktop: там меньше терминальных действий.

### 3. Получите свой MCP URL или token

1. На странице аккаунта найдите блок `MCP`, `API keys`, `Integrations`, `Connectors` или `Developer settings`.
2. Нажмите `Generate`, `Create`, `Add API key` или `Copy MCP URL`.
3. Скопируйте готовую ссылку или токен.
4. Сохраните данные в безопасном месте: password manager, личная защищенная заметка или локальный файл вне публичного репозитория.

Безопасный учебный пример:

```text
https://higgsfield.ai/mcp?mcp_token=ВАШ_ЛИЧНЫЙ_ТОКЕН
```

Нельзя вставлять в учебные материалы настоящий токен участника, преподавателя или команды.

## Способ А. Подключить Higgsfield в Claude web / Desktop

Этот способ подходит для первого подключения без терминала.

### Шаг 1. Откройте настройки Claude

1. Откройте [claude.ai](https://claude.ai/) или Claude Desktop.
2. Откройте профиль или меню настроек.
3. Найдите раздел `Settings`, `Customize` или `Connectors`.

### Шаг 2. Добавьте custom connector

1. Нажмите `Add connector` или `Add custom connector`.
2. В поле названия напишите `Higgsfield`.
3. В поле URL вставьте ваш MCP URL.
4. Нажмите `Add` или `Connect`.

Пример:

```text
Название: Higgsfield
URL: https://higgsfield.ai/mcp?mcp_token=ВАШ_ЛИЧНЫЙ_ТОКЕН
```

### Шаг 3. Подтвердите доступ

1. Если откроется окно входа, войдите в Higgsfield.
2. Подтвердите подключение.
3. Вернитесь в Claude и проверьте, что connector появился в списке.

Если вы работаете в командном тарифе, добавление connector может быть доступно только владельцу или администратору workspace.

## Способ Б. Подключить Higgsfield в Claude Code

Claude Code подключает remote MCP через терминальную команду. Для удаленного сервиса обычно используется transport `http`.

### Шаг 1. Откройте Claude Code в папке проекта

1. Запустите Claude Code.
2. Откройте рабочую папку проекта.
3. Откройте terminal, если команда вводится не в основном интерфейсе.

### Шаг 2. Добавьте Higgsfield MCP

Вставьте команду:

```bash
claude mcp add --transport http higgsfield "https://higgsfield.ai/mcp?mcp_token=ВАШ_ЛИЧНЫЙ_ТОКЕН"
```

### Шаг 3. Проверьте подключение

```bash
claude mcp list
```

Внутри Claude Code также можно открыть список MCP:

```text
/mcp
```

Если Claude Code просит пройти авторизацию в браузере, завершите вход и вернитесь в проект.

## Способ В. Подключить Higgsfield в Codex

В Codex подключение remote MCP зависит от версии Codex CLI и доступных настроек в вашей среде. Перед подключением проверьте, что команда `codex` доступна.

```bash
codex --version
```

Если ваша версия поддерживает добавление remote MCP через CLI, используйте команду:

```bash
codex mcp add higgsfield --url "https://higgsfield.ai/mcp?mcp_token=ВАШ_ЛИЧНЫЙ_ТОКЕН"
codex mcp list
```

Если команда недоступна, используйте один из рабочих вариантов:

- подключите Higgsfield в Claude / Claude Code;
- работайте в Higgsfield через web-интерфейс;
- используйте Codex для подготовки промптов, style rules, production brief и проверки результата.

## Шаг 1. Проверьте, что Higgsfield работает

После подключения откройте новый чат и попросите агента сделать тестовую генерацию.

```text
Сгенерируй через Higgsfield тестовое изображение.

Формат: 1:1.
Стиль: минималистичная editorial-обложка.
Сюжет: рабочий стол редактора, ноутбук, заметки, спокойный свет.
Запреты: без текста на изображении, без логотипов, без водяных знаков.

После генерации покажи результат или дай ссылку, где его открыть.
```

Проверьте:

- появился ли результат в чате;
- появилась ли ссылка на результат;
- сохранился ли результат в библиотеке Higgsfield;
- списались ли кредиты ожидаемым образом.

Если результата нет, попросите агента проверить статус:

```text
Проверь статус последней генерации Higgsfield.
Если результат готов, дай ссылку и коротко напиши:
1. какой формат получился;
2. где открыть файл;
3. как скачать его.
```

## Шаг 2. Генерируйте изображения внутри агента

Для изображения агенту нужно дать:

- формат;
- сюжет;
- визуальный стиль;
- композицию;
- ограничения;
- запреты.

Рабочий пример:

```text
Сгенерируй через Higgsfield изображение.

Формат: 9:16, вертикальная обложка для Reels / Shorts.
Сюжет: современная редакция регионального медиа, человек смотрит расшифровку видео и выделяет факты.
Стиль: чистый editorial, технологично, без чрезмерной корпоративности.
Композиция: внизу оставить свободное место под заголовок и плашку.
Цвета: черный, белый, красный акцент.
Запреты: без текста внутри картинки, без логотипов, без водяных знаков, без лишних рук и искаженных лиц.

Сделай 2 варианта и покажи, какой лучше подходит для обложки.
```

Если есть референс, загрузите его и уточните, что именно нужно использовать:

```text
Используй прикрепленную картинку как референс стиля.
Не копируй один в один.
Сохрани настроение: крупная типографика, спокойный фон, красный акцент, редакционный характер.
Сгенерируй новую картинку для темы: [вставьте тему].
Без текста внутри изображения.
```

Для обложек, карточек и плашек лучше не просить модель писать финальный текст внутри изображения. Текст должен оставаться редактируемым в Canva, Figma, HTML-конструкторе или монтажной программе.

## Шаг 3. Генерируйте видео внутри агента

Для видео задайте:

- длительность;
- формат;
- первый кадр или сцену;
- движение камеры;
- стиль;
- ограничения;
- запреты.

Начинайте с короткого теста на 5-8 секунд.

```text
Сгенерируй через Higgsfield короткое видео.

Длительность: 6 секунд.
Формат: 9:16.
Сцена: редакторский рабочий стол, на экране открыта расшифровка интервью, рядом заметки и маркер.
Движение камеры: медленный push-in к экрану ноутбука.
Стиль: реалистичный editorial, мягкий свет, спокойный технологичный вид.
Запреты: без читаемого текста на экране, без логотипов, без водяных знаков, без деформированных рук.

После генерации дай ссылку и напиши, подходит ли видео как фон для вертикальной обложки.
```

Если нужно видео из картинки:

```text
Используй прикрепленную картинку как первый кадр.
Сделай короткое видео 5 секунд, формат 16:9.
Движение: легкий кинематографичный zoom-in и мягкое движение света.
Не меняй композицию резко.
Не добавляй новых людей и лишние объекты.
Результат должен быть спокойным фоном для титра.
```

Если нужны варианты:

```text
Сделай 3 варианта этого видео:
1. спокойный editorial;
2. более динамичный social формат;
3. минималистичный фон под титр.

После генерации сравни варианты в таблице:
- стиль;
- где использовать;
- риски;
- какой выбрать.
```

## Как соединять изображения и видео

Для редакционной работы удобен маршрут:

```text
статичное изображение
-> короткое видео из изображения
-> текст и плашки отдельно
-> проверка человеком
```

### 1. Сделайте изображение-основу

Сначала создайте ключевой кадр. Проверьте композицию и оставьте место под титр или плашку.

### 2. Превратите изображение в видео

Загрузите изображение обратно в чат и попросите использовать его как первый кадр. Задайте одно движение: приближение, легкая панорама или движение света.

### 3. Добавьте текст отдельно

Финальный текст, плашки, имена, должности и цифры добавляйте в монтажной программе, Canva, Figma или визуальном конструкторе. Так их можно проверить и исправить перед публикацией.

Рабочий запрос:

```text
Сделай production route для этого ассета:
1. Сначала картинка 9:16 без текста.
2. Потом видео 6 секунд из этой картинки.
3. Потом отдельный список плашек, которые нужно наложить в монтажной программе.
4. Финальный текст не запекать в видео.

После каждого шага спрашивай подтверждение, прежде чем идти дальше.
```

## Готовые промпты

### Промпт для обложки

```text
Сгенерируй через Higgsfield обложку 16:9.
Тема: [вставьте тему].
Сцена: [что должно быть в кадре].
Стиль: [например: editorial, clean, cinematic, realistic].
Композиция: оставь свободное место слева для заголовка.
Запреты: без текста, без логотипов, без водяных знаков, без странных лиц и рук.
Сделай 2 варианта и предложи лучший.
```

### Промпт для видео-фона

```text
Сгенерируй через Higgsfield видео-фон.
Длительность: 6 секунд.
Формат: 9:16.
Сцена: [описание сцены].
Движение: медленный zoom-in / панорама / движение света.
Важно: фон должен быть спокойный, чтобы поверх можно было поставить текст.
Запреты: без текста внутри видео, без логотипов, без резких вспышек.
```

### Промпт для серии в одном стиле

```text
Сделай серию из 5 изображений через Higgsfield в одном визуальном стиле.
Общая тема: [вставьте тему].
Стиль: [описание стиля].
Формат: 1:1.
Каждая картинка должна быть отдельной сценой, но выглядеть как часть одной серии.
Без текста, без логотипов, без водяных знаков.
После генерации дай таблицу: номер, что изображено, где использовать.
```

## Если что-то не работает

### Connector не подключается

Проверьте:

- URL начинается с `https://`;
- вставлен весь MCP URL;
- токен не обрезан;
- вы вошли в правильный аккаунт Higgsfield;
- в Claude Code команда `/mcp` показывает сервер;
- в Codex команда `codex mcp list` показывает сервер, если ваша версия поддерживает MCP CLI.

Запрос агенту:

```text
Higgsfield MCP не подключается.
Проверь подключение и объясни простыми словами:
1. виден ли MCP-сервер;
2. нужна ли авторизация;
3. какая команда проверки;
4. что сделать дальше.
```

### Генерация идет долго

Видео обычно создается дольше изображения. Не запускайте много дублей подряд, пока первый запрос еще выполняется: это может расходовать кредиты.

```text
Проверь статус последней генерации Higgsfield.
Если она еще в работе, скажи это.
Если она завершилась, дай ссылку.
Если она упала, объясни причину и предложи более простой промпт.
```

### Результат не подходит

Не меняйте весь запрос сразу. Исправляйте одну проблему за один цикл: руки, фон, свет, композицию, движение камеры или степень реализма.

```text
Сделай новую версию, исправь только это:
[напишите проблему]

Все остальное сохрани:
- формат;
- композицию;
- стиль;
- свободное место под текст.

Не добавляй текст, логотипы и водяные знаки.
```

## Финальный чеклист

| Готово | Что проверить | Как понять, что все нормально |
|---|---|---|
| □ | Аккаунт | Вы вошли в Higgsfield и понимаете, где смотреть кредиты. |
| □ | Токен | У вас есть личный MCP token или MCP URL. |
| □ | Claude | Connector добавлен через Settings / Connectors или через `claude mcp add`. |
| □ | Codex | MCP добавлен через `codex mcp add`, если эта команда доступна в вашей версии. |
| □ | Тест | Сгенерировано простое тестовое изображение. |
| □ | Видео | Сгенерирован короткий тестовый ролик 5-8 секунд. |
| □ | Референсы | Понятно, как загрузить картинку и использовать ее как референс. |
| □ | Текст | Финальный текст и плашки не запекаются в изображение или видео. |
| □ | Скачивание | Вы знаете, где открыть и скачать результат. |

Рабочая формула:

```text
подключить MCP
-> сделать тестовое изображение
-> сделать короткое видео
-> скачать результат
-> наложить текст и плашки отдельно
-> проверить факты и права
```
