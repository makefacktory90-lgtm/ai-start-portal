# Как создать визуальный конструктор для контента

Статус: участнический гайд  
Для блока: "Видео, визуал и агентные пайплайны"  
Подходит для: редактор, SMM-специалист, продюсер, дизайнер, автор курса  
Главный результат: локальный HTML-конструктор, где можно менять формат, заголовок, стиль, цвета, плашки и экспортировать готовые карточки или обложки.

## Назначение

Визуальный конструктор - это небольшая HTML-страница или локальное приложение, где пользователь выбирает формат, стиль, цвета, пишет заголовок и получает готовую обложку, плашку, карточку или баннер.

Главная идея:

```text
сначала снять стиль
-> затем выбрать форматы
-> затем собрать интерфейс
-> затем проверить на реальных текстах
```

Если начать сразу с кода, можно получить красивую страницу, которая не решает редакционную задачу. Конструктор должен строиться не от внешнего эффекта, а от повторяемого рабочего сценария.

## Что нужно до старта

### 1. Откройте инструмент для работы

Можно использовать:

- Codex Desktop;
- Claude Code;
- другой агентный инструмент, который умеет создавать локальные HTML / CSS / JS файлы.

Создайте рабочую папку на компьютере, например:

```text
Desktop/visual-constructor
```

Для первого учебного конструктора достаточно обычных файлов:

```text
index.html
styles.css
script.js
```

Сервер, база данных и сложная сборка на первом шаге не нужны.

### 2. Соберите 5-10 визуальных примеров

Подойдут:

- скриншоты постов;
- обложки видео;
- сторис;
- титры;
- плашки;
- баннеры;
- презентационные слайды;
- карточки Telegram / VK / сайта.

Лучше брать реальные материалы бренда, редакции или автора, под которого создается конструктор. Если примеров мало, загрузите 2-3 самых точных и явно напишите агенту, что данных немного.

### 3. Решите, что должен делать конструктор

Для первого запуска выберите 2-3 формата.

Примеры:

- обложка урока;
- вертикальная карточка;
- square post;
- lower third;
- name plate;
- quote card;
- баннер страницы.

Не пытайтесь сразу собрать аналог Canva. Первый конструктор должен решать одну понятную задачу.

## Шаг 1. Снять визуальный стиль

Задача первого шага - получить style DNA: цвета, типографику, композицию, плашки, ритм, ограничения и правила повторения.

### 1. Загрузите визуальные примеры

1. Откройте Codex или Claude Code.
2. Прикрепите скриншоты или перетащите их в чат.
3. Если примеров много, сначала загрузите 5 самых характерных.

### 2. Используйте полный промпт для анализа визуального стиля

Используйте отдельный файл:

- [Промпт для анализа ДНК визуального стиля изображения](../../ai_start_course_site/assets/prompts/prompt-image-style-dna-analysis.md)

Ключевой фрагмент:

```text
You are an art director with 15 years of experience in fashion and advertising.

I'm uploading an image. Analyze its DNA like a professional.
```

На выходе нужно получить:

- detailed breakdown;
- recipe for series;
- one-line tags;
- ограничения, которые нельзя нарушать.

### 3. Превратите стиль в правила конструктора

После анализа стиля попросите агента перевести результат в параметры интерфейса.

```text
Теперь преврати этот визуальный стиль в технические правила для конструктора.

Сделай таблицу:
- параметр конструктора;
- какие варианты должны быть в интерфейсе;
- значение по умолчанию;
- что нельзя разрешать пользователю.

Пример параметров:
формат, цветовая тема, акцентный цвет, расположение текста, плашка, фон, логотип, экспорт.
```

Хороший результат - не описание "в красивом стиле", а набор конкретных правил: где заголовок, какие цвета, какие отступы, какие плашки, где нельзя размещать текст.

## Шаг 2. Выбрать форматы контента

Формат - это размер и назначение визуала. У конструктора обязательно должны быть конкретные форматы.

| Формат | Размер | Где используется | Что меняет пользователь |
|---|---:|---|---|
| Story | 1080 x 1920 | Stories, вертикальные карточки | заголовок, подзаголовок, фон, плашка |
| Square | 1080 x 1080 | Telegram, VK, посты | заголовок, цитата, логотип, цвет |
| YouTube | 1920 x 1080 | обложки видео и уроков | крупный заголовок, тема, speaker plate |
| Banner | 1920 x 640 | шапки страниц | короткий тезис, навигационная метка |
| Lower third | 1920 x 220 | нижние плашки в видео | имя, роль, тема выпуска |
| Quote bar | 1080 x 260 | цитаты и тезисы | цитата, автор, акцентный цвет |

Для первого учебного конструктора выберите максимум 3 формата. Если задача связана с видео, добавьте `Lower third`, `Name plate` или `Episode label`.

Запрос агенту:

```text
Помоги выбрать форматы для визуального конструктора.

Контент, под который делаем конструктор: [уроки, видео, Telegram, карусели, плашки].
Аудитория: абсолютные новички.
Стартовые форматы: Story 1080x1920, Square 1080x1080, Lower third 1920x220.

Сделай таблицу:
- формат;
- размер;
- где используется;
- какие поля должен менять пользователь.
```

## Шаг 3. Собрать задание для Codex или Claude Code

Когда у вас есть style DNA и список форматов, дайте агенту техническое задание.

### Вариант А. Если делаете в Codex

1. Откройте рабочую папку.
2. Вставьте задание ниже.
3. Добавьте style DNA и таблицу форматов.

```text
Собери локальный визуальный конструктор для контента.

Технологии: обычные HTML, CSS и JavaScript без сложной сборки.
Файлы: index.html, styles.css, script.js.

Интерфейс:
- слева панель настроек;
- в центре живой предпросмотр;
- справа экспорт и подсказки;
- форматы: [вставьте свои форматы и размеры];
- поля текста: заголовок, подзаголовок, kicker, name plate;
- настройки: тема, акцентный цвет, расположение текста, фон, сетка, плашки;
- экспорт PNG и SVG.

Важно:
- текст должен оставаться редактируемым;
- факты, цифры и цитаты нельзя запекать в AI-картинку;
- дизайн должен повторять этот визуальный стиль: [вставьте style DNA].

После сборки запусти локальную проверку и дай ссылку, которую можно открыть в браузере.
```

Если агент просит разрешение создать файлы в рабочей папке, это нормальное действие. Если агент предлагает удалить существующие файлы, сначала проверьте, что это не затронет нужные материалы.

### Вариант Б. Если делаете в Claude Code

Откройте Claude Code в рабочей папке и используйте то же задание. В конце добавьте:

```text
Сделай это в моей рабочей папке.
Не используй сложный фреймворк.
Создай простой локальный проект: HTML + CSS + JS.

После завершения напиши:
1. какие файлы созданы;
2. какой файл открыть в браузере;
3. как добавить новый формат;
4. как заменить цвета и тексты.
```

## Шаг 4. Проверить, что конструктор работает

### 1. Откройте файл в браузере

Откройте `index.html` или `visual-builder.html`. Если агент дал локальную ссылку вроде `http://127.0.0.1:4318`, откройте ее.

### 2. Проверьте основные действия

| Готово | Что проверить | Как понять, что все нормально |
|---|---|---|
| □ | Форматы | При переключении меняется размер холста. |
| □ | Текст | Заголовок меняется сразу после ввода. |
| □ | Цвета | Акцентный цвет меняет плашки или элементы. |
| □ | Плашки | Имя, роль или тема не вылезают за границы. |
| □ | Экспорт | PNG или SVG скачивается на компьютер. |
| □ | Стиль | Результат похож на исходные примеры, а не на случайный шаблон. |

Для учебного первого результата достаточно: настройки слева, предпросмотр в центре, экспорт и понятные поля ввода.

## Шаг 5. Добавить режим плашек

Плашки - отдельный тип контента. Их нельзя делать тем же макетом, что и пост или обложку. Им нужны короткие поля, стабильные отступы и защита от слишком длинного текста.

Запрос агенту:

```text
Добавь в конструктор отдельный режим для плашек.

Нужные форматы:
- Lower third: 1920x220;
- Name plate: 1080x180;
- Episode label: 1080x140;
- Quote bar: 1080x260.

Поля:
- имя или заголовок;
- роль или подзаголовок;
- короткая тема;
- акцентный цвет;
- логотип или текстовая метка.

Важно:
- текст не должен вылезать за границы;
- длинные строки нужно уменьшать или переносить;
- экспорт PNG и SVG должен работать;
- старые форматы конструктора нельзя ломать.
```

После добавления режима проверьте длинное имя, длинную должность, длинный заголовок и экспорт.

## Что делать, если что-то сломалось

### Страница не открывается

```text
Страница конструктора не открывается или пустая.
Проверь ошибки в HTML/CSS/JS, запусти локальную проверку и исправь.
После исправления объясни простыми словами, что было сломано.
```

### Дизайн не похож на референсы

```text
Сравни текущий конструктор с этими визуальными примерами.
Напиши, что не совпадает по стилю.
Потом внеси правки в CSS и JS, чтобы конструктор стал ближе к референсам.
Не меняй структуру проекта без необходимости.
```

### Текст вылезает за границы

```text
В некоторых форматах текст вылезает за границы.
Добавь защиту:
- перенос строк;
- ограничение ширины;
- уменьшение размера шрифта для длинного текста;
- проверку для мобильного и широкого формата.

Проверь на длинных русских заголовках.
```

## Human gate

Перед использованием конструктора в реальной публикации проверьте:

- правильно ли указаны имена, должности, даты, числа и география;
- не попал ли важный текст внутрь AI-картинки как не редактируемый слой;
- читается ли заголовок на мобильном экране;
- не нарушает ли стиль бренд или редакционные правила;
- есть ли права на изображения, видео и референсы;
- можно ли быстро исправить ошибку перед публикацией.

## Финальный чеклист

| Готово | Что проверить | Как понять, что все нормально |
|---|---|---|
| □ | Style DNA | Есть простые правила визуального стиля. |
| □ | Форматы | Выбраны 2-3 стартовых формата и размеры. |
| □ | Поля | Понятно, какие тексты пользователь меняет сам. |
| □ | Конструктор | Есть HTML/CSS/JS или аналогичный простой проект. |
| □ | Плашки | Добавлены отдельные размеры для плашек, если они нужны. |
| □ | Экспорт | PNG/SVG скачиваются и открываются. |
| □ | Проверка | Результат сравнен с исходными примерами. |
| □ | Инструкция | Пользователь понимает, какой файл открыть и что нажать. |

Главное правило:

```text
конструктор делается не от красоты,
а от повторяемой задачи
```

Сначала стиль, потом форматы, потом интерфейс. Если после гайда вы можете открыть файл, поменять текст и скачать картинку, базовый конструктор работает.
