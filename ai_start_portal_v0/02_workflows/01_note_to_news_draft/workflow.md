# Workflow 01: заметка -> черновик новости

Статус: approved starter  
Уровень: A1  
Владелец: TBD  
Версия: v0.1

## Когда использовать

Используй, если есть сырой материал:

- заметка корреспондента;
- расшифровка;
- пресс-релиз;
- сообщение от коллеги;
- факты с мероприятия;
- короткое интервью.

## Не использовать

Не используй для:

- расследований;
- конфликтных тем без редактора;
- юридически рискованных утверждений;
- публикации без проверки;
- материалов, где нет источника фактов.

## Стартовая команда

Скопируйте и вставьте в Codex / Claude Code:

```text
Открой AI Start Kit.
Используй workflow "01_note_to_news_draft".

Задача:
1. Сначала выдели только факты из материала.
2. Отдельно выпиши, что нужно проверить.
3. Потом сделай короткий черновик новости для сайта.
4. Потом сделай Telegram-версию.
5. Не выдумывай факты, имена, цифры, цитаты или источники.
6. Все сомнительное вынеси в блок "нужно проверить".
7. В конце дай список рисков для редактора.

Материал:
[вставьте материал ниже]
```

## Ожидаемый output

```text
## Факты из материала

- ...

## Нужно проверить

- ...

## Черновик новости

Заголовок:

Текст:

## Telegram-версия

...

## Риски / вопросы редактору

- ...
```

## Уровни задания

### A1: сделать безопасный черновик

Сделай только:

```text
факты -> нужно проверить -> черновик новости -> Telegram-версия -> риски
```

Цель:

- получить рабочий первый draft;
- не выдумать факты;
- понять, что нужно проверить.

### A2: адаптировать под роль / канал

После A1 попроси:

```text
Сделай 3 версии:
1. нейтральная новость для сайта;
2. короткая Telegram-версия до 900 знаков;
3. версия для анонса / соцсетей с более живым заходом.

Потом сравни, где меняется тон и что нельзя искажать.
```

Цель:

- увидеть разницу между каналами;
- не копировать один и тот же текст везде;
- научиться управлять тоном без потери фактов.

### A3: предложить улучшение workflow

Если AI ошибся или редактор постоянно правит одно и то же, попроси:

```text
Сделай update proposal для workflow.
Укажи:
- какая проблема повторяется;
- пример из output;
- что нужно добавить в инструкцию;
- как проверить изменение на 3 примерах;
- риск от изменения.
```

Цель:

- улучшить общий workflow;
- не чинить одну и ту же ошибку вручную каждую неделю.

## Quality gate

Перед отправкой редактору проверь:

- все факты есть в исходном материале;
- нет придуманных цитат;
- нет придуманных причин / последствий;
- есть блок `нужно проверить`;
- заголовок не обещает больше, чем есть в материале;
- Telegram-версия не искажает смысл;
- текст не звучит как пресс-релиз, если нужен новостной материал.

## Частые ошибки

| Ошибка | Что делать |
|---|---|
| AI добавил контекст из головы | Удалить и добавить в `нужно проверить` |
| Слишком рекламный тон | Попросить нейтральную новостную версию |
| Нет вопросов на проверку | Вернуть на шаг фактов |
| Слишком длинный Telegram | Попросить 700-900 знаков |
| Слишком общий заголовок | Попросить 5 вариантов с разным фокусом |

## Когда логировать проблему

Сохрани пример в `05_update_log`, если:

- AI повторно выдумывает один и тот же тип факта;
- workflow не подходит под реальный материал;
- чеклист не ловит важный риск;
- редактор постоянно правит одну и ту же вещь.
