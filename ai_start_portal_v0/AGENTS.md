# AI Start Kit Instructions

Эта папка - общий рабочий kit для редакционных AI workflows.

Она нужна не для того, чтобы "учить AI", а чтобы делать повторяющуюся редакционную работу быстрее и аккуратнее:

```text
материал -> workflow -> черновик -> проверка -> пример / проблема -> обновление системы
```

## Как работать

1. Начинайте с `00_start_here/`.
2. Выберите workflow из `02_workflows/`.
3. Используйте только факты из исходного материала или из утвержденного контекста.
4. Все сомнительное выносите в блок `нужно проверить`.
5. Перед отправкой редактору прогоните результат через `03_quality_gates/`.
6. Хорошие и плохие примеры сохраняйте в `04_examples/`.
7. Повторяющиеся проблемы заносите в `05_update_log/`.

## Главные правила

- AI output - это черновик, не публикация.
- Не выдумывайте факты, имена, цифры, цитаты, источники.
- Не скрывайте неопределенность. Пишите `нужно проверить`.
- Не меняйте approved workflows молча.
- Если один и тот же косяк повторяется 3+ раза, предложи update.
- Участники логируют примеры; process owners обновляют shared workflows.

## Для новичка

Если вы не понимаете, что выбрать, используйте:

```text
02_workflows/01_note_to_news_draft/workflow.md
```

Это базовый сценарий:

```text
заметка / событие / расшифровка -> факты -> вопросы на проверку -> черновик новости -> Telegram-версия
```

## Уровни заданий

В каждом workflow могут быть три уровня.

```text
A1 базовый
Используйте готовый workflow, получите результат, проверьте и сохраните пример.

A2 средний
Адаптируйте результат под канал, аудиторию или свою роль.

A3 продвинутый
Найдите повторяющуюся проблему и предложите update для команды.
```

Правило:

> A1 - это не слабый уровень. Это нормальный рабочий минимум.

Если вы корреспондент, устали, у вас материал и дедлайн, делайте A1.

Если A1 получился спокойно, берите A2.

Если вы видите, что AI системно ошибается, берите A3 или приносите пример в office hours.

## Где что лежит

```text
00_start_here/       вход для участников
01_context/          утвержденный контекст и правила редакции
02_workflows/        рабочие сценарии
03_quality_gates/    чеклисты перед редактором / публикацией
04_examples/         approved / rejected / before-after примеры
05_update_log/       проблемы, предложения, changelog
06_process_owners/   материалы для владельцев workflows
```

## Формат имен файлов

Используйте дату и понятное название:

```text
2026-05-14_tv21_note_to_news_example_01.md
2026-05-14_severny_gorod_rejected_telegram_example_02.md
```

## Когда звать человека

Зовите редактора / руководителя, если:

- факт влияет на репутацию, безопасность, деньги, право или людей;
- есть цитата без источника;
- AI сделал вывод, которого нет в материале;
- текст может быть опубликован наружу;
- есть сомнения в тоне или формулировке.
