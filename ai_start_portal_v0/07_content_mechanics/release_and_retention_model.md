# Механика выдачи контента и удержания

Статус: starter v0.1  
Назначение: описывает, как участник получает материалы, что делает до / во время / после созвона и как курс не умирает после последней встречи.

## 1. Принцип

Каждый блок курса должен быть устроен одинаково.

Участник не должен каждый раз разбираться, где запись, где промпт, где домашка и куда нести результат. Если человек ищет кнопку, мы уже проигрываем внимание.

## 2. Структура страницы блока

```text
01_before/
  what_this_is.md
  prework.md
  materials_to_prepare.md

02_live/
  deck.pdf / deck.pptx
  demo_script.md
  workflow.md
  practice_round.md

03_after/
  recording.md
  transcript.md
  summary.md
  homework_A1_A2_A3.md
  feedback_log_link.md
  next_step.md
```

## 3. До встречи

Открывается за 24-48 часов.

Содержит:

- зачем блок;
- что подготовить;
- один короткий prework;
- workflow, который будем использовать;
- предупреждение о рисках.

Формат prework:

```text
10-20 минут.
Достаточно выполнить базовый минимум.
Если не успели - приходите все равно.
```

## 4. Во время встречи

Структура 90-120 минут:

```text
0-10    зачем это в работе
10-25   live demo
25-55   практика в группах
55-75   разбор good / messy / risky outputs
75-90   quality gate + сохранение evidence
```

Группы:

- operator;
- editor;
- recorder.

## 5. После встречи

Открывается в день встречи.

Содержит:

- запись;
- транскрипт;
- краткое резюме;
- starter prompt;
- домашку A1/A2/A3;
- место для результата;
- feedback log.

## 6. Домашка

### A1

Сделать один рабочий output по готовому workflow.

### A2

Адаптировать output под канал / формат / аудиторию.

### A3

Найти проблему и предложить update для hub.

## 7. Удержание

Удерживаем не мотивацией, а маленькими обязательствами.

### Минимум участника

- один прогон материала через workflow в неделю;
- один сохраненный output;
- одна отметка "что было трудно".

### Минимум редакции

- один короткий разбор output на планерке;
- один пример в approved / rejected examples;
- один AI-hub owner смотрит feedback log.

### Минимум проекта

- office hours / case clinic;
- weekly changelog;
- обновление prompts только через evidence;
- не больше 1-2 новых правил за неделю.

## 8. После финала

30-дневный adoption loop:

| Неделя | Вопрос | Артефакт |
|---|---|---|
| 1 | Где оно раздражает и ломается? | friction log |
| 2 | Что реально получилось? | 5 approved examples |
| 3 | Что надо закрепить? | updated workflows / gates |
| 4 | Что оставляем, меняем, убиваем? | keep / revise / kill board |

## 9. Где будут записи

На каждой странице блока должны быть слоты:

```text
Запись Zoom: [добавить после встречи]
Транскрипт: [добавить после встречи]
Резюме: [добавить после встречи]
Материалы участников: [ссылка]
```

До появления записи слот остается видимым, но помеченным как "появится после встречи".
