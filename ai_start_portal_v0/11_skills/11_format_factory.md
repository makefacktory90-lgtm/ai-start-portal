# 11 Format Factory

## Назначение

Собрать из одного Source Map несколько форматов: сайт, Telegram, social, карточки, заголовки, spoken script. Факты остаются теми же, задача канала меняется.

## Когда использовать

- есть один source или Source Map;
- нужно быстро подготовить пакет для разных площадок;
- SMM и сайт должны работать от одной фактической базы;
- нужно избежать кликбейта и потери контекста.

## Основной prompt

```text
На основе Source Map и Style Graph собери package форматов.

Форматы:
1. Сайт: заголовок, лид, структура, что проверить.
2. Telegram: первый экран, главный факт, причина продолжить чтение.
3. Social: одна мысль без кликбейта.
4. Карточки: 5-7 экранов, только проверяемые факты.
5. Audio/spoken: 60-90 секунд, текст можно произнести вслух.
6. 5 заголовков с разными фокусами.

Правила:
- не добавляй фактов вне Source Map;
- канал меняет задачу, а не факты;
- сохраняй Style Graph;
- все сомнительное вынеси в "нужно проверить";
- не делай финальную публикацию там, где нужен редактор.

Source Map:
[вставить]

Style Graph:
[вставить или написать "нет, собрать draft"]
```

## Output format

```text
## Format strategy
| формат | задача | что сохраняется | что меняется | риск |

## Сайт
## Telegram
## Social
## Карточки
## Audio / spoken
## Заголовки
## Нужно проверить
```

## Format rules

- сайт объясняет;
- Telegram дает быстрый вход;
- social держит одну мысль;
- карточки делают последовательность;
- audio должно нормально произноситься;
- заголовки проверяют фокус, а не обманывают аудиторию.

## Human gate

- все форматы сверены с Source Map;
- нет новых причин, оценок, цифр, цитат;
- кликбейт удален;
- Style Graph применен не декоративно, а по evidence;
- редактор выбирает, что выпускать.
