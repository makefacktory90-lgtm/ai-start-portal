# 14 Style Graph Protocol

Status: draft for lesson 02. Based on Ira's `style-graph-protocol.md`. Do not publish before 15 May.

## Назначение

Собирать и выращивать Style Graph не за один промпт, а через evidence, выборы пользователя и decision traces.

Style Graph - это не красивое описание голоса. Это живая карта решений:

- что автору близко;
- что звучит чужим;
- какие приемы повторяются;
- какие механики запрещены;
- какие правила подтверждены примерами;
- что пока остается гипотезой.

## Три режима без обязательного агента

### 1. Один чат

Подходит для старта. Пользователь сохраняет Style Graph прямо в диалоге и периодически просит обновить snapshot.

Минус: легко потерять контекст.

### 2. Проект / папка

Подходит для нормальной работы. В папке лежат:

- хорошие примеры;
- плохие примеры;
- Style Card;
- Style Graph snapshot;
- anti-style;
- approved/rejected outputs;
- changelog.

### 3. Skill

Подходит для продвинутого уровня. Повторяемый протокол упаковывается в `SKILL.md`, references и examples.

## Стартовый prompt

```text
Начинаем сборку Style Graph.

Я буду давать вам примеры текстов и реакции на них.
Не собирай финальный портрет после одного примера.
Работай через evidence и мои подтверждения.

На каждом шаге:
1. Покажите, какие механики видны в тексте.
2. Дай evidence: короткие цитаты или фрагменты.
3. Предложи мне выбрать:
   - близко;
   - привлекательно, но не мое;
   - точно не брать;
   - нужно проверить на других примерах.
4. Сохраняй только подтвержденные сигналы.
5. Обновляй decision traces.

Первый текст:
[вставьте текст, который нравится]
```

## Decision trace format

```text
## Decision Trace
- Signal:
- Evidence:
- User decision: keep / reject / tension / test later
- Rule candidate:
- Confidence: low / medium / high
- Next test:
```

## Обновление Style Graph

```text
Обнови Style Graph на основе нового текста и моей реакции.

Не перезаписывай старые правила без причины.
Раздели:
1. Stable rules.
2. New signals.
3. Rejected mechanics.
4. Tensions.
5. What to test next.

Новый материал:
[текст]

Моя реакция:
[что нравится / не нравится / что точно мое / что чужое]
```

## Human gate

- не превращать один пример в правило;
- не сохранять гипотезу как stable rule;
- не описывать автора прилагательными вместо механизмов;
- не копировать чужой стиль поверх своего;
- сохранять rejected mechanics так же внимательно, как approved.

## Домашка Base / Opti / Advanced

- Base: один хороший текст + выбор близких механик.
- Opti: хороший текст, плохой текст, свой текст, первый Style Graph snapshot.
- Advanced: вести decision traces и подготовить skill candidate.
