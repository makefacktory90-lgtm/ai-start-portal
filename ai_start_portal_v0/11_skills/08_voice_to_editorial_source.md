# 08 Voice To Editorial Source

Status: draft for lesson 02. Do not publish before 15 May.

## Назначение

Превратить голосовую диктовку, хаотичную заметку или расшифровку мысли в рабочую редакционную заготовку: source, факты, вопросы, форматы, риски.

Это не режим "сделать красивый текст из любого потока речи". Голосовая диктовка становится только исходным материалом, который нужно структурировать и проверить.

## Когда использовать

- корреспондент надиктовал заметки после события;
- редактор проговорил идею материала;
- SMM записал голосом мысль для поста;
- после Zoom / созвона нужно вытащить задачи;
- участник не хочет печатать первый черновик.

## Что нужно до старта

- диктовка из Wispr Flow / Superwhisper / Handy / встроенной диктовки;
- понимание, для какого материала это нужно;
- запрет на публикацию без source check.

## Основной prompt

```text
Я надиктовал(а) сырой поток мыслей.

Не пиши публикацию.
Сначала преврати это в рабочую редакционную заготовку.

Сделай:
1. Главная мысль.
2. Факты, которые прозвучали.
3. Предположения и оценки отдельно от фактов.
4. Что нужно проверить.
5. Какие форматы возможны: сайт, Telegram, social, карточки, видео, аудио.
6. Что спросить у редактора перед текстом.
7. Где диктовка звучит непонятно или обрывается.

Если мысль обрывается или не хватает данных, не додумывай. Пометь: "нужно уточнить".

Диктовка:
[вставить текст]
```

## Output format

```text
## Главная мысль
## Факты из диктовки
## Оценки / предположения
## Нужно проверить
## Возможные форматы
## Вопросы редактору
## Что уточнить у автора
```

## Human gate

- диктовка не считается источником факта сама по себе;
- имена, даты, должности, числа проверяются отдельно;
- эмоциональные оценки не превращаются в утверждения;
- если человек "наговорил красиво", это все равно не публикация;
- перед текстом нужен Source Map.

## Домашка Base / Opti / Advanced

- Base: надиктовать 3 минуты сырого материала и получить структурированную заготовку.
- Opti: из заготовки собрать Source Map и один безопасный draft.
- Advanced: сохранить собственный voice-to-source prompt как shortcut в рабочей папке.
