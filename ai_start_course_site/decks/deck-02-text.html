// TEXT OS

Материал
становится системой текстовых форматов

Сегодня не разбираем “как написать текст в AI”. Разбираем рабочий способ: взять источник, извлечь факты, удержать голос, пересобрать материал под каналы и проверить output перед выпуском.

source EXO formats QA hub update
// СДВИГ

Не просим AI “написать”. Сначала просим увидеть материал.

Текст начинается не с красивого абзаца, а с чистого входа.

Если AI не видит факты, он начинает уверенно дорисовывать.

Если нет gate, гладкий текст выглядит готовым раньше времени.

Поэтому первый навык — извлечение, а не генерация.

// СХЕМА УРОКА

Четыре коротких цикла практики внутри занятия

01EXO: разобрать источник
02Smell test: убрать AI-гладкость
03Repurpose: собрать форматы
04QA: проверить риски
05Hub: сохранить правило
// DATA LAYER

EXO превращает сырой материал в карту фактов

factsчто точно есть в источнике
peopleлюди, организации, места, должности
quotesкто что сказал и можно ли это цитировать
missingчего не хватает для уверенной публикации
risksгде возможны юридические, тональные или репутационные ошибки
// PRACTICE 01 · 10 МИН

Упражнение: разобрать материал до первого текста

03 мин. Вставьте общий source в starter prompt.

04 мин. Проверьте EXO: что есть, чего не хватает, где риск.

03 мин. Отметьте один факт, который нельзя усиливать или додумывать.

// VOICE LAYER

Голос редакции — это не настроение, а набор проверяемых решений

ритм: короткие / длинные фразы, плотность, темп.

словарь: какие слова звучат как редакция, а какие нет.

доказательства: как показываем факт, цитату, число, контекст.

запреты: где AI начинает звучать рекламно, драматично или канцелярски.

// PRACTICE 02 · 8 МИН

Smell test:
найти AI-гладкость до публикации

01. Найдите 3 фразы, которые можно поставить в любой текст.

02. Замените каждую фразу на факт, цитату или конкретное наблюдение.

03. Сформулируйте одно правило для Style Graph.

// SKILL LAYER

Repurpose — это не “укоротить”, а сменить задачу формата

сайтконтекст, факты, спокойная структура
Telegramбыстрый вход, смысл, проверяемый акцент
socialодна мысль, первый экран, без кликбейта
карточкитолько факты в редактируемом слое
// PRACTICE 03 · 12 МИН

Собрать publication package из одного source

04 мин. Черновик для сайта: заголовок, лид, структура.

04 мин. Telegram-версия: первый экран, факт, зачем читать.

04 мин. Social-анонс и 5 заголовков с разным фокусом.

// PROMPT AS WORKFLOW

Один сильный prompt держит источник, формат и проверку

Задача: собрать publication package.

1. Не пишите финал, пока не сделан EXO.
2. Все сомнительное вынесите в "нужно проверить".
3. Для каждого формата укажите цель канала.
4. Не усиливайте смысл, если этого нет в source.
5. В конце дайте QA-gate и update proposal.
// METRICS / GATES

Качество текста проверяется до красоты

factualнет фактов, которых не было в источнике
toneтекст не звучит как пресс-релиз без редакции
riskсомнительное вынесено в вопросы редактору
formatканал меняет подачу, а не факты
// DATA SAFETY

В Text OS есть материалы, которые не загружаются в общий AI

greenпубличные источники и учебные материалы
yellowвнутренние черновики только по разрешенной политике
redперсональные данные, доступы, договоры, непубличные конфликты
// PRACTICE 04 · 8 МИН

QA-проверка: что нельзя выпускать без человека

01. Найдите один факт, который требует проверки.

02. Найдите одну фразу, где AI усилил смысл.

03. Решите: правим prompt, context, Style Graph или gate.

// ROLE LAYER

Один pipeline дает разные полезные outputs для разных ролей

корреспондент

редактор

// FLYWHEEL

Система становится сильнее, когда хорошие и плохие outputs сохраняются

01source

// HUB ACTION

После занятия остается не впечатление, а артефакт
