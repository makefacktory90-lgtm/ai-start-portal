// SOURCE MAP

Source Map — это разбор источника до первого текста

factsчто точно есть в источнике
peopleлюди, организации, должности, роли
quotesцитаты и можно ли их использовать
missingчего не хватает для уверенной публикации
riskгде AI может усилить, сгладить или додумать
// PROMPT 01 · SOURCE MAP

Первый промпт: собрать карту материала, а не текст

Разбери материал как редакционный источник.

Не пиши финальный текст.

Сначала задай до 5 уточняющих вопросов, если без них нельзя выбрать формат.
Если вопросов нет, сразу собери Source Map.

Таблица Source Map:
1. Факт / утверждение из источника.
2. Где это видно в источнике: цитата, таймкод, абзац, документ.
3. Кто действует: человек, организация, роль.
4. Что можно сказать уверенно.
5. Что нельзя усиливать, обобщать или додумывать.
6. Какой формат подходит: сайт, Telegram, соцсети, карточки, видео.
7. Вопрос редактору перед публикацией.

Если чего-то нет в источнике, пиши: "нет в источнике".
// STYLE GRAPH

Style Graph — это не «tone of voice красивыми словами»

rhythmдлина фраз, темп, паузы, плотность
lexiconсвои слова, чужие слова, запрещенная пластмасса
evidenceкак текст доказывает мысль: факт, сцена, цитата, число
stanceнасколько текст объясняет, спорит, поддерживает, предупреждает
anti-styleчто сразу убивает голос редакции
// PROMPT 02 · STYLE GRAPH DRAFT

Второй промпт: вытащить правила голоса из хороших примеров

Проанализируй примеры хороших текстов редакции.

Собери Style Graph:

1. Ритм: как устроены фразы и абзацы.
2. Словарь: какие слова и конструкции повторяются.
3. Доказательства: как текст опирается на факт, место, героя, цитату, число.
4. Запреты: какие фразы звучали бы чужеродно.
5. Anti-style: признаки пластикового AI-текста.
6. 7 правил, по которым нужно проверять новый output.

Не оценивай вкус абстрактно. Дай наблюдения с примерами.

// PROMPT 03 · DEPLASTICIZE

Третий промпт: не переписать красиво, а убрать пластик

Проверь текст на пластиковую AI-гладкость.

Найди:
1. Универсальные фразы, которые подходят к любому материалу.
2. Раздувание события.
3. Уверенные утверждения без источника.
4. Канцелярский или рекламный тон.
5. Симметричные абзацы, которые звучат одинаково.

Для каждого места предложи замену через:
факт / цитату / место / героя / число / конкретное последствие.

Не переписывай весь текст сразу. Сначала покажи проблемные места и риск.

// PROMPT 04 · PUBLICATION PACKAGE

Четвертый промпт: собрать пакет форматов без потери фактов

На основе Source Map и Style Graph собери publication package.

Форматы:
1. Сайт: заголовок, лид, структура, что проверить.
2. Telegram: первый экран, главный факт, зачем читать.
3. Social: короткий анонс без кликбейта.
4. 5 заголовков с разным фокусом.

Правила:
- не добавляй факты, которых нет в Source Map;
- не усиливай масштаб события;
- сохраняй Style Graph;
- все сомнительное вынеси в "нужно проверить".

// QA

Качество текста проверяется до красоты

factualнет фактов, которых не было в источнике
toneтекст не звучит как общий пресс-релиз
riskсомнительное вынесено в вопросы редактору
formatканал меняет подачу, а не факты
// PROMPT 05 · QA GATE

Финальный промпт: проверить перед человеком, а не вместо человека

Проверь publication package перед редактором.

Сделай таблицу:
1. Факт / утверждение.
2. Есть ли это в источнике?
3. Риск: факт, тон, контекст, юридический, репутационный.
4. Что должен проверить человек.
5. Что нужно убрать или переписать.

Не утверждай, что текст готов к публикации.
Финальное решение принимает редактор.
