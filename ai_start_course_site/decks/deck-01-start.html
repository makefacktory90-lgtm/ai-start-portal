// AGENT LAYER

У AI-системы три слоя: думать, вести процесс, делать операции.

LLM layer

ChatGPT / Claude / Gemini

Помогают думать, писать, разбирать источник, искать формулировки и видеть слабые места.

Agent layer

Codex / Claude Code

Ведут рабочую папку: prompts, gates, examples, source map, outputs и обновления hub.

Production layer

Higgsfield / OpusClip / Submagic / Gling

Делают конкретные операции: визуал, клипы, субтитры, нарезку, обложки и черновой production.

// ПОРТАЛ

Портал нужен, чтобы хорошие приемы не оставались в личном чате.

После встреч здесь лежат записи, промпты, проверки, примеры и маршруты, которые можно повторить в работе.

проверитьфакты, контекст, тон и что можно сохранить для команды
// ВОПРОСЫ РЕДАКЦИИ

Ключевые вопросы превращаем в рабочие gates.

а)Как ловить AI-клише и шаблонные ошибки?

smell_test_prompt.md: возвращаем текст к факту, человеку, месту и редакционному смыслу.

б)Как защищать внутренние данные?

source_safety_gate.md: разделяем материалы на green / yellow / red до загрузки в AI.

в)Может ли AI навредить изданию?

publication_gate.md: проверяем уникальность, пользу, факты и ответственность перед выпуском.

практикаГде будет практика?

На встрече общий прогон, между встречами свой материал, вопросы и outputs в чат.

// SAFETY GATE

Data gate: сначала классифицируем материал, потом решаем, можно ли нести его в AI.

greenпубличные материалы, утвержденные тексты, открытые источники
yellowвнутренние черновики без персональных данных и чувствительных деталей
redперсональные данные, доступы, договоры, непубличные документы, расследования
// PUBLICATION GATE

Publication gate: опасен не AI, а дешевый неуникальный output без редакционной ценности.

originalесть собственный факт, герой, место или наблюдение редакции
verifiedключевые факты проверены, спорные места отмечены
usefulматериал отвечает читателю, а не просто заполняет канал
ownedесть человек, который отвечает за выпуск
// ПЕРВЫЙ НАВЫК

Researcher ищет не ссылки, а проверяемый материал для редакционной работы.

SCOUT: найти источники с разных углов: факты, контекст, сомнения.

CRITIC: отсеять слабые, рекламные и непроверяемые источники.

METHOD: зафиксировать, как искали и почему доверяем.

ACTION: превратить найденное в editorial source map.

GATE: что человек обязан проверить перед публикацией.

// ПРАКТИКА · УСТАНАВЛИВАЕМ RESEARCHER

02
