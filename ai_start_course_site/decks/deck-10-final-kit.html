// PROMPT BASE

Базовый принцип: сильный промпт - это не магическая формула, а короткое рабочее задание.

Для редакции, маркетинга, HR и аналитики меняются задачи, но не меняется каркас: роль, задача, условия, ограничения, формат результата и human gate.

// PROMPT DNA

Базовая ДНК промпта для ТВ-команды

ролькто помогает: редактор, аналитик, маркетолог, HR, ресерчер, продюсер

задачачто именно нужно сделать и зачем это нужно команде

условиякакой вход даем: заметка, расшифровка, вакансия, бриф, таблица, ссылки, ограничения

ограничениячто нельзя выдумывать, что проверяет человек, какой формат ответа нужен

// PROMPT BOOSTERS

Как усиливать промпт после первой версии

Ты помогаешь мне как [роль / функция].

Задача:
[что нужно сделать и зачем].

Входные данные:
[что я даю].

Сделай:
1. [главный результат]
2. [дополнительный результат]
3. [что отметить как риск / пробел / вопрос]

Формат ответа:
[таблица / список / memo / steps].

Важно:
- не выдумывай факты;
- если данных не хватает, пометь это отдельно;
- не пиши лишнюю теорию;
- опирайся только на входные данные и явно отмечай допущения.

Выберите 1-2 усилителя под задачу:
1. задай 3 уточняющих вопроса;
2. оцени черновик по шкале 1-10;
3. предложи 5 способов усилить результат;
4. подключи 2 временных экспертов;
5. сделай pre-mortem или post-mortem;
6. покажи 2-3 варианта структуры;
7. выдели риски и пробелы;
8. выдай финальную версию только после self-check.
// PROMPT PRACTICE

Упражнение: любой слабый запрос можно докрутить до рабочего промпта.

шаг 1

Берем сырую задачу: например, “подготовить темы для HR-контента на июнь”.

шаг 2

Заполняем окна: роль, задача, условия, ограничения, усилитель, human gate.

шаг 3

Просим систему задать вопросы, оценить результат по шкале 1-10 и предложить улучшения.

вывод

Команда учится не “просить красиво”, а собирать повторяемые рабочие формулировки.

// EDITORIAL BEST CASE

Редакция: как выглядит хорошее внедрение

задача

один повторяемый участок

Например: еженедельный дайджест, публикационный пакет, адаптация для Telegram, набор заголовков или разбор мониторинга.

шаблон

не один чат, а структура

Есть карта источника, черновик, фактологическая проверка, правила стиля и понятная ручная проверка до публикации.

примеры

команда учится

Сохраняются удачные примеры, неудачные AI-результаты и правки, которые реально усилили результат.

результат

устойчивый эффект

Редактор тратит меньше времени на пустой черновик и больше времени на решение, что выпускать.

// MARKETING BEST CASE

Маркетинг: как выглядит хорошее внедрение

аудитория

ясный портрет

Есть исследование целевой аудитории, язык аудитории, барьеры, анти-аудитория и критерии выбора.

планирование

кампания как система

Есть бриф кампании: цель, сегмент, сообщения, каналы, контент-календарь, ответственный и метрики.

креатив

не вкус, а критерии

Идеи и креатив проходят через понятную оценку: уместность, конкретика, ясность, риски и ожидаемый эффект.

результат

следующий шаг ясен

Команда понимает, какие темы и форматы усиливать, а какие не дают результата и должны быть исключены.

// HR BEST CASE

HR: как выглядит хорошее внедрение

use case

конкретные задачи

Вакансии, HR-бренд, ответы кандидатам, контент-план, сравнение формулировок, FAQ по найму.

quality

ясность вместо канцелярита

AI помогает упрощать язык, убирать общие слова и делать коммуникацию более понятной для кандидата.

review

человек держит рамку

HR проверяет смысл, соответствие роли, tone of voice и репутационные риски перед использованием.

result

устойчивость

Команда получает повторяемые шаблоны, а не каждый раз начинает с нуля новый текст.

// ANALYTICS BEST CASE

Аналитика: как выглядит хорошее внедрение

input

таблицы и сигналы

На вход идут данные, заметки, логи мониторинга, списки тем, гипотезы, сравнения периодов.

prompt

анализ с рамкой

Промпт задает вопрос, формат ответа, критерии оценки и запрет на необоснованные выводы.

output

не “умный текст”, а решение

На выходе: таблица, summary, ranking, gaps, next actions, risks, assumptions и что нужно проверить отдельно.

result

польза для команды

Аналитик быстрее собирает первое объяснение картины и освобождает время на проверку и интерпретацию.

// 3-MONTH ROLLOUT

Трехмесячный rollout: не расширять хаотично, а идти по фазам

месяц 1

База: 1 повторяющаяся текстовая задача на отдел, prompt DNA, human gate, 3-5 прогонов, approved / rejected examples.

месяц 2

Стабилизация: доработка формулировок, quality gate, role-based prompts для маркетинга, HR, редакции, аналитики.

месяц 3

Расширение: repurposing, visual / audio / campaign formats, review того, что масштабировать следующим шагом.

всегда

Сохранять evidence: вход, промпт, результат, ручную правку, вывод и owner note.

// DEPARTMENT LOOP

Одинаковая логика внедрения для разных отделов

step 1

выбрать участок

Каждый отдел выбирает один конкретный повторяющийся тип работы, а не “все процессы сразу”.

step 2

добиться устойчивости

Сделать несколько прогонов подряд и добиться полезного повторяемого результата.

step 3

сохранить в систему

Сохранить prompt, example, failure, ручную правку и правило, которое улучшило результат.

step 4

переходить дальше

Только после устойчивого результата брать следующий участок работы или новый формат.

// PROJECTS + INFRA

Проекты разделяем по папкам и правилам, инфраструктуру обсуждаем отдельным техническим контуром

Для каждого проекта нужен свой рабочий контур. Общие привычки можно держать наверху, но продуктовые и редакционные правила должны жить внутри конкретного проекта.

projectsотдельная папка, собственный `CLAUDE.md`, локальные примеры и правила применения

shared layerтолько нейтральные рабочие настройки, без продуктовой специфики

infraотдельный технический контур: Hetzner, DigitalOcean, OVHcloud, Akamai Cloud (Linode)

specialistподрядчика и специалиста лучше рекомендовать отдельно, после дополнительного подтверждения

// STRATEGY + NEXT

Финал курса - это не закрытие темы, а запуск управляемого цикла внедрения.

На выходе у команды должна остаться не “память о лекциях”, а рабочая система: prompt base, pilot cases по отделам, owners, weekly review, examples и понятный следующий трехмесячный шаг.
