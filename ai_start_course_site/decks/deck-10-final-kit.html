// PROMPT BASE

Базовый принцип: сильный промпт - это не магическая формула, а короткое рабочее задание.

Для редакции, маркетинга, HR и аналитики меняются задачи, но не меняется каркас: роль, задача, условия, ограничения, формат результата и human gate.

// PROMPT DNA

Базовая ДНК промпта для ТВ-команды

ролькто помогает: редактор, аналитик, маркетолог, HR, ресерчер, продюсер

задачачто именно нужно сделать и зачем это нужно команде

условиякакой вход даем: заметка, расшифровка, вакансия, бриф, таблица, ссылки, ограничения

ограничениячто нельзя выдумывать, что проверяет человек, какой формат ответа нужен

// PROMPT BOOSTERS

Как усиливать промпт после первой версии

Ты помогаешь мне как [роль / функция].

Задача:
[что нужно сделать и зачем].

Входные данные:
[что я даю].

Сделай:
1. [главный результат]
2. [дополнительный результат]
3. [что отметить как риск / пробел / вопрос]

Формат ответа:
[таблица / список / memo / steps].

Важно:
- не выдумывай факты;
- если данных не хватает, пометь это отдельно;
- не пиши лишнюю теорию;
- опирайся только на входные данные и явно отмечай допущения.

Усили промпт:
1. задай 3 уточняющих вопроса;
2. оцени черновик по шкале 1-10;
3. предложи 5 способов усилить результат;
4. подключи 2 временных экспертов;
5. сделай pre-mortem и post-mortem;
6. покажи 2-3 варианта структуры;
7. выдели риски и пробелы;
8. выдай финальную версию только после self-check.
// PROMPT PRACTICE

Упражнение: любой слабый запрос можно докрутить до рабочего промпта.

шаг 1

Берем сырую задачу: например, “подготовить темы для HR-контента на июнь”.

шаг 2

Заполняем окна: роль, задача, условия, ограничения, усилитель, human gate.

шаг 3

Просим систему задать вопросы, оценить результат по шкале 1-10 и предложить улучшения.

вывод

Команда учится не “просить красиво”, а собирать повторяемые рабочие формулировки.

// MARKETING PIPELINES

Что именно полезно маркетингу корпоративной компании

Определение целевой аудитории, buyer persona, язык боли, критерии выбора и ограничения по сегментам.

// 3-MONTH ROLLOUT

Логика внедрения: сначала тексты, потом repurposing, потом review и расширение

месяц 1

Текстовый контур: 3-5 типовых задач, единый prompt guide, approved / rejected examples, weekly review.

месяц 2

Расширение в форматы: visual brief, audio script, repurposing под каналы, quality gates для новых форматов.

месяц 3

Retention и анализ: what worked, what failed, чего не хватает, что масштабировать, что исключить.

постоянно

Сохранять evidence: вход, промпт, результат, ручную правку, вывод и owner note.

// OWNERS + CRITERIA

Что должно остаться после финальной встречи

owners

2-3 человека

Ответственный за качество результата, ответственный за hub, ответственный за сбор практики и обновлений.

pilot cases

3 кейса

Один редакционный, один маркетинговый, один HR- или аналитический сценарий на первый цикл.

review loop

раз в неделю

Сохраняем, дорабатываем, исключаем или откладываем автоматизацию.

success

понятные признаки

Появились approved / rejected examples, улучшен хотя бы один prompt, команда понимает следующий шаг.

// PROJECTS + INFRA

Проекты разделяем по папкам и правилам, инфраструктуру обсуждаем отдельным техническим контуром

Для каждого проекта: отдельная папка, собственный `CLAUDE.md`, локальные примеры и правила применения. На общий уровень выносятся только нейтральные рабочие настройки. По инфраструктуре можно рассматривать провайдеров вроде Hetzner, DigitalOcean, OVHcloud и Akamai Cloud (Linode), а выбор подрядчика и специалиста целесообразно делать отдельно, после дополнительного подтверждения и согласования с IT и безопасностью.

// STRATEGY + NEXT

Финал курса - это не закрытие темы, а запуск управляемого цикла внедрения.

На выходе у команды должна остаться не “память о лекциях”, а рабочая система: prompt base, marketing / HR / editorial pilot cases, owners, weekly review, examples и следующий трехмесячный шаг.
