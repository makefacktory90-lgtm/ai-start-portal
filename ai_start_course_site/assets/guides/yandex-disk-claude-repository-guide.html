Как работать с репозиторием редакции на Яндекс.Диске через Claude

Для блока: "AI-hub и репозитории ТВ-21"

Подходит для: редактор, выпускающий редактор, продюсер, SMM-специалист, владелец AI-hub

Результат: у редакции есть рабочая папка-репозиторий на Яндекс.Диске, которую можно открыть как проектную папку в Claude / Claude Code / Codex Work и использовать для хранения материалов, базовых промптов, рабочих промптов, примеров и обновлений.

Назначение

Репозиторий редакции нужен, чтобы материалы курса и рабочие AI-практики хранились в единой управляемой структуре, а не в личных чатах, случайных заметках и отдельных файлах.

Предлагаемая схема работы:

Яндекс.Диск редакции
-> папка "Репозиторий редакции"
-> синхронизированная папка на компьютере
-> Claude / Claude Code / Codex Work открывает эту папку как рабочий проект
-> редакция обновляет промпты, примеры и правила по единой структуре

Важно: в этом сценарии Яндекс.Диск используется как синхронизированная рабочая папка. Claude или Codex работают с файлами в этой папке, а Яндекс.Диск синхронизирует изменения для команды.

Что нужно до старта

### 1. Доступ к Яндекс.Диску

Нужен аккаунт Яндекс.Диска и папка, к которой есть доступ у ответственных сотрудников.

Для первого запуска лучше использовать отдельную папку:

Репозиторий редакции

Не рекомендуется начинать с общей папки, где уже лежат все исходники редакции. Сначала создается чистая структура, затем в нее переносятся необходимые материалы.

### 2. Синхронизация Яндекс.Диска с компьютером

Чтобы Claude / Claude Code / Codex Work могли работать с файлами, папка должна быть доступна на компьютере как обычная локальная папка.

Проверьте:

  • установлен Яндекс.Диск для компьютера;
  • нужная папка видна в Finder / проводнике;
  • файлы открываются без браузера;
  • изменения в папке синхронизируются обратно в Яндекс.Диск.

Если папка доступна только через веб-интерфейс, агент не сможет работать с ней как с полноценным проектом. Веб-интерфейс подходит для загрузки файлов, но не для регулярной работы с репозиторием.

### 3. Рабочая среда

Можно использовать один из вариантов:

  • Claude Desktop / Claude web с загрузкой файлов и проектными материалами;
  • Claude Code, если он установлен и умеет открывать локальную папку проекта;
  • Codex Work / Codex Desktop, если команда работает через локальную папку;
  • другой агентный инструмент, который может читать и редактировать файлы в выбранной папке.

Для владельца репозитория удобнее Claude Code или Codex Work, потому что они работают с папкой как с проектом: читают структуру, создают файлы, обновляют промпты и фиксируют изменения.

Шаг 1. Создайте папку репозитория на Яндекс.Диске

В Яндекс.Диске создайте папку:

Репозиторий редакции

Внутри создайте базовую структуру:

Репозиторий редакции/
  00_start_here/
  01_sources/
  02_base_prompts/
  03_working_prompts/
  04_examples/
  05_quality_gates/
  06_guides/
  07_updates/
  08_archive/

### Что означает каждая папка

| Папка | Что хранится |

|---|---|

| 00_start_here | правила входа, карта папок, роли, инструкция для нового участника |

| 01_sources | исходные материалы: тексты, ссылки, транскрипты, заметки, source maps |

| 02_base_prompts | стабильные промпты курса, которые не меняются каждый день |

| 03_working_prompts | рабочие промпты редакции, которые обновляются по практике |

| 04_examples | удачные и проблемные примеры: input, AI output, правка редактора |

| 05_quality_gates | проверки перед публикацией: факт, тон, видео, аудио, визуал, безопасность |

| 06_guides | инструкции для команды: как открыть, как обновлять, как подключать |

| 07_updates | журнал изменений: что обновили, почему, кто владелец |

| 08_archive | старые версии, которые нельзя удалять, но не нужно использовать ежедневно |

Не используйте абстрактные названия папок вроде разное, новое, для ИИ, важное. Такие названия быстро затрудняют поиск материалов, контроль версий и передачу процесса между сотрудниками.

Шаг 2. Разделите базовые и рабочие промпты

В репозитории должны быть два разных слоя промптов.

### Базовые промпты

Папка:

02_base_prompts/

Здесь лежат стабильные промпты, которые команда получила на курсе.

Примеры:

02_base_prompts/
  01_editorial_digest_prompt.md
  02_release_plan_prompt.md
  03_editorial_task_prompt.md
  04_voiceover_prompt.md
  05_video_texture_prompt.md
  06_visual_style_dna_prompt.md

Правило: базовый промпт не переписывается после одного неудачного результата. Если требуется адаптация, сначала создается рабочая версия.

### Рабочие промпты

Папка:

03_working_prompts/

Здесь лежат промпты, которые редакция адаптирует под свои рубрики, тон, источники и повторяющиеся задачи.

Пример структуры:

03_working_prompts/
  text/
    news_to_telegram_working.md
    source_to_site_news_working.md
  video/
    long_video_to_clips_working.md
    transcript_to_story_map_working.md
  audio/
    material_to_voiceover_working.md
  planning/
    weekly_editorial_digest_working.md
    release_plan_working.md

Правило: рабочие промпты можно обновлять, но каждое заметное изменение должно попадать в 07_updates.

Шаг 3. Создайте стартовые README-файлы

В папке 00_start_here создайте файл:

README.md

Текст можно взять такой:

# Репозиторий редакции

Это рабочая папка для AI-процессов редакции.

Здесь хранятся:
- исходные материалы;
- базовые промпты;
- рабочие промпты;
- примеры;
- quality gates;
- гайды;
- журнал обновлений.

Главное правило:
AI помогает готовить черновики, карты материала, варианты и проверки.
Финальное редакционное решение принимает человек.

В папке 03_working_prompts создайте файл:

README.md

Пример:

# Рабочие промпты

Здесь лежат промпты, которые редакция использует в текущей работе.

Обновляем их по evidence:
- повторяющиеся ошибки;
- удачные outputs;
- правки редактора;
- новые рубрики;
- новые ограничения;
- изменения в workflow.

Не обновляем промпт только на основании субъективного впечатления.
Нужно указать пример и причину.

Шаг 4. Откройте папку репозитория в Claude / Codex Work

### Вариант А. Claude Code

1. Откройте Claude Code.

2. Выберите Open folder или откройте терминал в папке репозитория.

3. Укажите локальный путь к папке Яндекс.Диска.

Пример пути на macOS:

/Users/имя_пользователя/Yandex.Disk/Репозиторий редакции

Пример пути на Windows:

C:\Users\имя_пользователя\YandexDisk\Репозиторий редакции

Названия папок могут отличаться. Ориентируйтесь на фактическое место, где Яндекс.Диск синхронизирует файлы на вашем компьютере.

После открытия папки попросите Claude проверить структуру:

Проверь структуру этой папки как репозитория редакции.

Нужно:
1. Описать, какие папки уже есть.
2. Найти, каких обязательных папок не хватает.
3. Предложить минимальную структуру для AI-hub редакции.
4. Ничего не удалять.
5. Если нужно создать недостающие README, сначала покажи список файлов.

### Вариант Б. Codex Work / Codex Desktop

1. Откройте Codex.

2. Выберите рабочую папку.

3. Укажите папку Репозиторий редакции внутри Яндекс.Диска.

4. Попросите Codex прочитать структуру и предложить стартовую настройку.

Стартовый запрос:

Открой эту папку как репозиторий редакции.

Задача:
- проверить структуру;
- создать недостающие README;
- разнести базовые и рабочие промпты;
- подготовить журнал обновлений;
- не удалять существующие файлы;
- все изменения описывать перед внесением.

### Вариант В. Claude web без локального проекта

Если команда пока работает только в браузерном Claude, можно начать проще:

1. Открыть папку Яндекс.Диска в браузере.

2. Скачать нужные файлы или папку.

3. Загрузить их в Claude как проектные материалы.

4. Попросить Claude подготовить структуру и новые файлы.

5. После проверки вручную загрузить обновленные файлы обратно на Яндекс.Диск.

Этот вариант подходит для обучения, но менее удобен для регулярной работы. Ручные загрузки повышают риск расхождения версий.

Шаг 5. Дайте агенту правила работы с репозиторием

В корне репозитория создайте файл:

AGENTS.md

Пример содержимого:

# Правила работы с репозиторием редакции

1. Не удалять файлы без отдельного подтверждения.
2. Не переносить исходные материалы в публичные папки.
3. Не добавлять персональные данные, доступы, договоры и непубличные документы в AI без проверки.
4. Базовые промпты хранить отдельно от рабочих промптов.
5. Рабочие промпты обновлять только по evidence: пример, ошибка, правка, повторяющаяся задача.
6. Каждое обновление промпта фиксировать в `07_updates/changelog.md`.
7. Перед публикацией использовать quality gates.
8. Если назначение файла непонятно, сначала уточнить у владельца, а не переименовывать без согласования.

Этот файл задает границы работы агента и снижает риск случайного удаления, переименования или неконтролируемого изменения материалов.

Шаг 6. Настройте журнал обновлений

В папке 07_updates создайте файл:

changelog.md

Стартовый шаблон:

# Журнал обновлений репозитория

## 2026-05-26

### Что изменили
- Создали структуру репозитория.
- Разделили базовые и рабочие промпты.
- Добавили папки для examples, quality gates и guides.

### Почему
- Нужно отделить стабильные материалы курса от рабочих промптов редакции.

### Кто владелец
- [имя / роль]

### Что проверить через месяц
- Какие промпты использовались в работе.
- Какие промпты нужно упростить.
- Какие рабочие промпты дали полезный результат.
- Какие ошибки повторяются.

Шаг 7. Ежемесячно обновляйте папку рабочих промптов

Минимальный цикл обновления:

1 раз в месяц
-> открыть 3-5 реальных кейсов
-> сравнить input / AI output / правку редактора
-> найти повторяющиеся ошибки
-> обновить рабочий промпт
-> записать изменение в changelog
-> сохранить пример в examples

Промпт для ежемесячного обновления:

Проведи ежемесячное ревью рабочих промптов редакции.

Папки:
- 03_working_prompts
- 04_examples
- 05_quality_gates
- 07_updates

Сделай:
1. Найди рабочие промпты, которые использовались чаще всего.
2. Найди повторяющиеся ошибки по примерам.
3. Отдельно выпиши, где проблема в промпте, а где в исходных данных.
4. Предложи обновления только для тех промптов, где есть evidence.
5. Для каждого обновления укажи:
   - какой файл меняем;
   - что меняем;
   - почему;
   - на какой пример опираемся;
   - какой риск снимаем.
6. Подготовь запись для `07_updates/changelog.md`.

Не переписывай базовые промпты.
Не удаляй старые версии.
Сначала покажи план изменений.

Шаг 8. Как хранить примеры

В папке 04_examples создайте подкаталоги:

04_examples/
  approved/
  needs_review/
  rejected/

Каждый пример лучше хранить одним .md-файлом:

2026-05-26_news_to_telegram_approved.md

Шаблон:

# Пример: [название]

Дата:
Формат:
Канал:
Владелец:
Состояние: approved / needs_review / rejected

## Source

[что было на входе]

## Prompt

[какой промпт использовали]

## AI output

[что получилось]

## Правка редактора

[что изменил человек]

## Вывод для репозитория

- что сработало;
- что надо запретить;
- какой prompt / gate обновить.

Шаг 9. Что нельзя класть в репозиторий без проверки

Перед загрузкой материалов в папку, с которой работает AI, проверьте безопасность.

Не кладем без отдельного решения:

  • персональные данные;
  • доступы, пароли, токены, приватные ссылки;
  • договоры;
  • непубличные финансовые документы;
  • юридически чувствительные материалы;
  • внутренние расследования;
  • материалы с ограничениями по правам;
  • документы, которые нельзя передавать внешним AI-сервисам.

Если сомневаетесь, используйте отдельную папку:

01_sources/_needs_security_review/

И не просите агента обрабатывать эти файлы, пока владелец не разрешил.

Шаг 10. Проверка, что система работает

Через неделю после запуска проверьте:

| Проверка | Да / нет |

|---|---|

| У команды есть единая ссылка на папку Яндекс.Диска |

| Папка синхронизируется на компьютер владельца |

| Claude / Codex открывает папку как рабочий проект |

| Есть 00_start_here/README.md |

| Базовые промпты отделены от рабочих |

| Есть папка 03_working_prompts |

| Есть журнал 07_updates/changelog.md |

| Есть хотя бы один approved example |

| Есть хотя бы один needs_review или rejected example |

| Понятно, кто владелец обновлений |

Если значительная часть пунктов не выполнена, репозиторий пока не работает как устойчивая система. В этом случае нужно завершить настройку структуры, ролей и правил обновления.

Короткий сценарий для первого занятия

1. Создаем папку "Репозиторий редакции" на Яндекс.Диске.
2. Внутри создаем 9 базовых папок.
3. Синхронизируем папку на компьютер.
4. Открываем эту папку в Claude / Codex Work.
5. Просим агента проверить структуру.
6. Кладем базовые промпты в `02_base_prompts`.
7. Создаем первые рабочие промпты в `03_working_prompts`.
8. Создаем `07_updates/changelog.md`.
9. Договариваемся о ежемесячном ревью рабочих промптов.

Минимальный результат

После настройки у редакции должно быть:

  • одно место для материалов AI-практики;
  • понятное разделение базовых и рабочих промптов;
  • папка для примеров;
  • quality gates;
  • журнал изменений;
  • владелец обновлений;
  • правило ежемесячного пересмотра рабочих промптов.

Это минимальный рабочий репозиторий: понятная структура папок, правила обновления, примеры, quality gates и ответственный владелец процесса.
